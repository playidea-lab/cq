# C4 설치 가이드

이 문서는 C4를 설치하고 Claude Code에서 사용할 수 있도록 설정하는 방법을 안내합니다.

C4는 **Go MCP 서버**(Primary, 148 tools: 118 base + 30 Hub)와 **Python Sidecar**(LSP, Document Parsing, 10 tools)로 구성됩니다.

## 사전 요구사항

### 필수
- **Go 1.22+**: Go MCP 서버 빌드에 필요
- **Python 3.11+**: Sidecar(LSP, C2 Document Parsing)에 필요
- **uv**: Python 패키지 관리자 ([설치 가이드](https://docs.astral.sh/uv/getting-started/installation/))
- **Claude Code**: Anthropic의 AI 코딩 어시스턴트

### 선택
- **Node.js 18+ / pnpm / Rust toolchain**: C1 데스크톱 앱 빌드 시 필요
- **jq**: `.mcp.json` 자동 병합에 사용 (없으면 Python 폴백)

## 빠른 설치

### 원라인 설치 (권장)

```bash
curl -sSL https://raw.githubusercontent.com/PlayIdea-Lab/cq/main/install.sh | bash
```

자동으로 `~/c4`에 클론 후 빌드합니다. 커스텀 경로를 지정하려면:

```bash
C4_INSTALL_DIR=/opt/cq curl -sSL https://raw.githubusercontent.com/PlayIdea-Lab/cq/main/install.sh | bash
```

### 클론 후 설치

```bash
git clone https://github.com/PlayIdea-Lab/cq.git && cd cq
./install.sh
```

### install.sh가 자동으로 처리하는 항목

1. Go / Python / uv 버전 확인
2. (원격 실행 시) 소스 자동 클론 — 이미 존재하면 `git pull`로 업데이트
3. Go 바이너리 빌드 (`c4-core/bin/cq`)
4. Python 의존성 설치 (`uv sync`)
5. `.mcp.json` 설정 (기존 파일 보존, jq 또는 Python 폴백)
6. `.c4/` 디렉토리 초기화
7. (대화형 터미널) `~/.local/bin/cq`에 글로벌 설치 여부 확인

> 비대화형 환경에서 글로벌 설치를 원하면: `C4_GLOBAL_INSTALL=y curl ... | bash`

설치 후 **Claude Code를 재시작**하면 148개 MCP 도구가 등록됩니다 (Hub 비활성 시 118개).

## 수동 설치

### 1. 소스 클론

```bash
git clone https://github.com/PlayIdea-Lab/cq.git
cd cq
```

### 2. Go 바이너리 빌드

```bash
cd c4-core && go build -o bin/cq ./cmd/c4/ && cd ..
```

빌드 결과: `c4-core/bin/cq` (~12MB)

#### 글로벌 설치 (선택)

모든 프로젝트에서 C4를 사용하려면 `~/.local/bin/`에 설치합니다:

```bash
cd c4-core && go build -o ~/.local/bin/cq ./cmd/c4/ && cd ..
```

> **주의**: macOS ARM64에서는 반드시 `go build -o`를 사용하세요. `cp`로 복사하면 코드 서명이 무효화됩니다.

### 3. Python Sidecar 의존성 설치

```bash
uv sync
```

Go MCP 서버가 필요 시 Python Sidecar를 자동으로 기동합니다 (Lazy Start). `uv sync`로 의존성만 설치하면 됩니다.

## MCP 서버 설정

### 프로젝트 설정 (기본)

프로젝트 루트의 `.mcp.json`이 Go MCP 서버를 구성합니다:

```json
{
  "mcpServers": {
    "cq": {
      "command": "/path/to/cq/c4-core/bin/cq",
      "args": ["mcp", "--dir", "/path/to/cq"],
      "env": {
        "C4_PROJECT_ROOT": "/path/to/cq"
      }
    }
  }
}
```

> **중요**: `/path/to/cq`를 실제 C4 설치 경로로 변경하세요. `install.sh`를 사용하면 자동으로 설정됩니다.

### 글로벌 설정 (모든 프로젝트에서 사용)

`~/.claude.json` 파일에 추가:

```json
{
  "mcpServers": {
    "cq": {
      "command": "~/.local/bin/cq",
      "args": ["mcp"]
    }
  }
}
```

### Claude Code 재시작

설정을 적용하려면 Claude Code를 **완전히 종료 후 다시 시작**합니다.

## Shell 자동완성

`cq -t <Tab>`으로 저장된 세션 이름을 자동완성할 수 있습니다.

### 자동 설치

**install.sh** 실행 시와 **`cq init`** 실행 시 자동으로 설정됩니다.
`~/.zshrc` 또는 `~/.bashrc`에 다음 한 줄이 추가됩니다:

```bash
# zsh
eval "$(cq completion zsh)"

# bash
eval "$(cq completion bash)"
```

### 수동 설치

자동 설치가 되지 않은 경우 직접 추가합니다:

```bash
# zsh (~/.zshrc에 추가)
echo 'eval "$(cq completion zsh)"' >> ~/.zshrc && source ~/.zshrc

# bash (~/.bashrc에 추가)
echo 'eval "$(cq completion bash)"' >> ~/.bashrc && source ~/.bashrc

# fish
cq completion fish >> ~/.config/fish/completions/cq.fish
```

### 사용법

```bash
cq -t <Tab>           # 저장된 세션 목록
cq claude -t <Tab>    # 동일
```

## OS 서비스 등록 (선택)

`cq serve`를 부팅 시 자동 시작하도록 OS 서비스로 등록할 수 있습니다.

### 자동 설치

`cq init` 실행 시 프롬프트가 표시됩니다:

```
Install cq serve as OS service (auto-start on boot)? [y/N]
```

### 수동 설치

```bash
cq serve install    # 서비스 등록
cq serve status     # 상태 확인
cq serve start      # 시작
cq serve stop       # 중지
cq serve uninstall  # 서비스 해제
```

> **macOS**: `~/Library/LaunchAgents/cq-serve.plist`에 설치됩니다 (sudo 불필요).
> **Linux**: systemd user service로 등록됩니다.

## 로그인 (필수)

C4 클라우드 기능을 사용하려면 로그인이 필요합니다. Supabase URL/Key는 바이너리에 내장되어 있으므로 별도 설정이 불필요합니다:

```bash
cq auth login      # 브라우저에서 GitHub OAuth 인증
cq auth status     # 인증 상태 확인
```

> **참고**: Supabase 크리덴셜은 빌드 시 바이너리에 내장됩니다. 사용자가 URL이나 키를 직접 설정할 필요가 없습니다.

## API 키 설정 (선택 — LLM Gateway 사용 시)

LLM Gateway(`c4_llm_call`, Knowledge 임베딩 등)를 사용하려면 API 키를 설정합니다.

### 권장: cq secret (암호화 저장)

```bash
cq secret set openai.api_key      # 입력값 echo 없음 (히스토리 미노출)
cq secret set anthropic.api_key
cq secret list                     # 저장된 키 이름 확인
```

저장 위치: `~/.c4/secrets.db` (AES-256-GCM 암호화, 전역 공유)
CI 환경: `C4_MASTER_KEY=<64 hex chars>` 환경변수로 복호화 키 주입

### config.yaml에서 LLM Gateway 활성화

```yaml
# .c4/config.yaml
llm_gateway:
  enabled: true
  default: openai
  providers:
    openai:
      enabled: true
      default_model: gpt-4o-mini
    # api_key는 cq secret set openai.api_key 로 저장 (자동 조회)
```

**키 해석 우선순위**: `~/.c4/secrets.db` (`cq secret set`) > 환경변수 (예: `ANTHROPIC_API_KEY`)

**Cloud 자동 활성화**: `config.yaml`의 `cloud.url` + `cloud.anon_key` 설정 시 자동 활성화됩니다.

## 문제 해결

### "MCP server not found" 오류

1. `.mcp.json` 또는 `~/.claude.json`에서 바이너리 경로 확인
2. Go 바이너리가 존재하는지 확인: `ls c4-core/bin/cq`
3. 바이너리가 없으면 다시 빌드: `cd c4-core && go build -o bin/cq ./cmd/c4/`
4. Claude Code 재시작

### macOS ARM64 코드 서명 오류

`cp`로 바이너리를 복사하면 코드 서명이 무효화됩니다. 반드시 `go build -o` 직접 빌드를 사용하세요:

```bash
# ✗ 잘못된 방법
cp c4-core/bin/cq ~/.local/bin/cq

# ✓ 올바른 방법
cd c4-core && go build -o ~/.local/bin/cq ./cmd/c4/
```

### Python Sidecar 오류

Go 서버가 Sidecar 기동에 실패할 경우:

1. `uv sync`로 Python 의존성 재설치
2. Python 3.11+ 설치 확인: `python3 --version`
3. uv 설치 확인: `uv --version`

> Python Sidecar는 Lazy Start로 동작합니다 — 첫 LSP/C2 도구 호출 시에만 기동됩니다. Python 미설치 시 LSP/C2 도구만 비활성화되고 나머지 102개 Go native 도구는 정상 작동합니다.

### Go 빌드 실패

1. Go 1.22+ 설치 확인: `go version`
2. 모듈 의존성 다운로드: `cd c4-core && go mod download`
3. 빌드 오류 확인: `cd c4-core && go build ./...`

### LLM Gateway 도구 미등록

LLM Gateway 도구(`c4_llm_call` 등)는 LLM API 키가 환경에 설정되어 있을 때만 등록됩니다. `.env`에 `ANTHROPIC_API_KEY` 또는 `OPENAI_API_KEY`를 추가하세요.

### MCP 도구가 작동하지 않음

1. `.mcp.json`에 cq MCP 서버가 등록되어 있는지 확인
2. MCP 서버 바이너리 경로가 올바른지 확인
3. Claude Code 완전 재시작

## 업데이트

C4를 최신 버전으로 업데이트하려면:

```bash
cd /path/to/cq
git pull

# Go 바이너리 재빌드
cd c4-core && go build -o bin/cq ./cmd/c4/ && cd ..

# 글로벌 바이너리도 설치한 경우
cd c4-core && go build -o ~/.local/bin/cq ./cmd/c4/ && cd ..

# Python 의존성 업데이트
uv sync
```

그 다음 Claude Code를 재시작합니다.

## 제거

C4를 제거하려면:

```bash
# 1. ~/.claude.json에서 "cq" MCP 서버 항목 삭제

# 2. (선택) 글로벌 바이너리 삭제
rm -f ~/.local/bin/cq

# 3. (선택) C4 폴더 삭제
rm -rf /path/to/cq
```
