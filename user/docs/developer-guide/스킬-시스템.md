# C4 스킬 시스템

이 문서는 C4의 스킬 시스템 아키텍처와 확장 방법을 설명합니다.

## 개요

스킬 시스템은 에이전트가 보유할 수 있는 **원자적 역량(Atomic Capability)**을 정의합니다. 태스크의 특성에 따라 필요한 스킬을 매칭하고, 해당 스킬을 가진 에이전트를 자동으로 선택합니다.

### 핵심 개념

```
┌─────────────────────────────────────────────────────────────┐
│                    스킬 계층 구조                            │
├─────────────────────────────────────────────────────────────┤
│ Layer 3: External Registry (옵션)                           │
│   github:vercel-labs/agent-skills                          │
│   npm:@c4-skills/react-optimization                        │
├─────────────────────────────────────────────────────────────┤
│ Layer 2: Project Skills (.c4/skills/)                       │
│   프로젝트 특화 스킬, 팀 커스텀 스킬                          │
├─────────────────────────────────────────────────────────────┤
│ Layer 1: Built-in Skills (C4 Core)                          │
│   c4/supervisor/agent_graph/skills/                         │
│   검증된 핵심 스킬 (debugging, testing, ...)                 │
└─────────────────────────────────────────────────────────────┘
```

## 스킬 스키마 (V2)

### 기본 구조

```yaml
skill:
  # === 필수 필드 ===
  id: debugging                    # 고유 ID (kebab-case)
  name: "Debugging & Error Analysis"
  description: "버그 식별, 루트 원인 분석..."
  capabilities:                    # 역량 목록
    - error-tracing
    - stack-analysis

  # === 트리거 (최소 1개 필수) ===
  triggers:
    keywords: [bug, error, crash]  # 키워드 매칭
    task_types: [debug, fix-bug]   # 태스크 타입 매칭
    file_patterns: ["*.log"]       # 파일 패턴 매칭

  # === V2 신규: Impact 우선순위 ===
  impact: critical | high | medium | low  # 기본: medium
  category: performance | security | quality | architecture | data-processing | operations

  # === V2 신규: 다중 도메인 ===
  domains: [web-frontend, ml-dl]   # 기본: [universal]
  domain_specific:
    ml-dl:
      priority_boost: 0.5          # 해당 도메인에서 가중치

  # === V2 신규: 메타데이터 ===
  metadata:
    version: "1.0.0"
    author: "c4-team"
    license: MIT
    tags: [debugging, error-analysis]

  # === V2 신규: 임베디드 규칙 ===
  rules:
    - id: DBG-001
      description: "문제 재현 단계를 먼저 정의한 후 디버깅 시작"
      impact: high
      example_bad: |
        # 재현 단계 없이 바로 코드 수정
      example_good: |
        # 1. 재현 단계 정의
        # 2. 최소 재현 케이스 작성

  # === V2 신규: 의존성 ===
  dependencies:
    required: [data-validation]    # 필수 의존
    optional: [profiling]          # 선택 의존

  # === 관계 필드 ===
  tools: [debugger, profiler]
  complementary_skills: [profiling, logging]
  leads_to: [testing, code-review]
```

### Impact 기반 우선순위 스코어링

```
score = impact_weight × (1 + domain_boost) + rule_bonus

- impact_weight: critical=2.0, high=1.5, medium=1.0, low=0.5
- domain_boost: 0~2 (도메인 매칭 시 domain_specific.priority_boost)
- rule_bonus: critical 규칙 수 × 0.1 (최대 0.5)
```

**예시**:
```
deployment 태스크:
  - deployment 스킬: 2.0 × (1 + 0.15) = 2.30 (critical + domain boost)
  - debugging 스킬: 1.5 × 1.0 = 1.50 (high, domain 불일치)
```

## CLI 명령어

### 스킬 목록 조회

```bash
# 기본 스킬 목록
c4 skill list

# 특정 도메인 필터
c4 skill list --domain ml-dl

# 프로젝트 스킬 포함
c4 skill list --dir .c4/skills

# deprecated 포함
c4 skill list --all
```

### 스킬 검증

```bash
# 단일 파일 검증
c4 skill validate skills/debugging.yaml

# 디렉토리 검증
c4 skill validate skills/

# 의존성 검사 포함
c4 skill validate skills/ --check-deps

# 상세 출력
c4 skill validate skills/ --verbose
```

### 스킬 정보 조회

```bash
c4 skill info debugging
c4 skill info experiment-tracking --dir c4/supervisor/agent_graph/skills
```

## 검증 레벨

### Level 1: 필수 (에러)
- JSON Schema 통과
- Pydantic 모델 통과
- 최소 1개 트리거 (keywords OR task_types OR file_patterns)
- 최소 1개 capability

### Level 2: 권장 (경고)
- rules 최소 1개 권장
- description 50자 이상 권장
- example_bad/example_good 쌍 권장

### Level 3: 선택 (정보)
- dependencies 검증 (`--check-deps`)
- 순환 의존성 검사

## 스킬 그룹

프로젝트 유형에 따라 스킬을 그룹으로 활성화할 수 있습니다.

```yaml
# skills/_groups.yaml
groups:
  core-essential:
    skills: [code-review, error-handling, debugging, testing]
    description: "Core skills every project needs"

  ml-essential:
    skills: [experiment-tracking, data-analysis, feature-engineering]
    description: "Essential skills for ML/DL projects"

  security-basic:
    skills: [security-scanning, authentication, error-handling]
    description: "Basic security skills for production"
```

**프로젝트 설정** (`.c4/config.yaml`):

```yaml
skills:
  enabled_groups:
    - ml-essential
    - security-basic
  disabled:
    - deployment  # 이 프로젝트에서 불필요
```

### 1. 프로젝트 스킬 추가

```bash
mkdir -p .c4/skills
```

```yaml
# .c4/skills/our-api-standards.yaml
skill:
  id: our-api-standards
  name: "Our API Standards"
  description: "회사 내부 API 설계 표준 및 네이밍 컨벤션"

  impact: high
  category: architecture
  domains: [web-backend, api]

  metadata:
    version: "1.0.0"
    author: "backend-team"
    tags: [api, standards, internal]

  capabilities:
    - api-naming-convention
    - error-response-format
    - pagination-standard

  triggers:
    keywords: [api, endpoint, rest]
    task_types: [api-design, api-implement]
    file_patterns:
      - "**/api/**"
      - "**/routes/**"

  rules:
    - id: API-001
      description: "모든 API 엔드포인트는 /api/v{version}/ 접두사 사용"
      impact: high
      example_bad: |
        GET /users/123
      example_good: |
        GET /api/v1/users/123

    - id: API-002
      description: "에러 응답은 표준 형식 사용"
      impact: critical
      example_good: |
        {
          "error": {
            "code": "USER_NOT_FOUND",
            "message": "User not found",
            "details": {}
          }
        }
```

### 2. 검증

```bash
c4 skill validate .c4/skills/our-api-standards.yaml
```

## SKILL.md 포맷 (Vercel 호환)

외부 SKILL.md 파일을 자동으로 파싱합니다.

```markdown
---
name: react-best-practices
description: 40+ React/Next.js optimization rules
impact: critical
domains: [web-frontend]
---

## When to Use
- React 컴포넌트 작성 시
- 번들 사이즈 최적화 시

## Rules

### PERF-001: Eliminate async waterfalls (impact: critical)

**Bad:**
```javascript
const user = await getUser();
const posts = await getPosts(user.id);
```

**Good:**
```javascript
const [user, posts] = await Promise.all([
  getUser(),
  getPosts(userId)
]);
```
```

## API 사용법

### 스킬 로딩

```python
from c4.supervisor.agent_graph import (
    ExternalSkillLoader,
    ExternalLoaderConfig,
    SKILLS_DIR,
)

# 기본 로더
config = ExternalLoaderConfig(builtin_path=SKILLS_DIR)
loader = ExternalSkillLoader(config)
result = loader.load_all()

print(f"Loaded {len(result.skills)} skills")
```

### 스킬 매칭

```python
from c4.supervisor.agent_graph import SkillMatcher, TaskContext

matcher = SkillMatcher(graph)
task = TaskContext(title="Fix deployment pipeline", task_type="deploy")
matches = matcher.extract_required_skills_with_scores(task)

for match in matches:
    print(f"{match.skill_id}: {match.total_score:.2f}")
```

### 스킬 검증

```python
from c4.supervisor.agent_graph import SkillValidator

validator = SkillValidator()
result = validator.validate_file(Path("skills/debugging.yaml"))

if result.is_valid:
    print("Validation passed")
else:
    for issue in result.issues:
        print(f"[{issue.level}] {issue.message}")
```
