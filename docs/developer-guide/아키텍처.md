## 핵심 컴포넌트

### C4Daemon

프로젝트 상태와 태스크 오케스트레이션을 관리하는 중앙 컨트롤러.

```python
class C4Daemon:
    def __init__(
        self,
        project_root: Path | None = None,
        state_store: StateStore | None = None,  # DI
    ):
        self.root = project_root or Path.cwd()
        self.state_machine = StateMachine(self.c4_dir, store=state_store)

    def initialize(self, project_id: str) -> C4State: ...
    def load(self) -> None: ...

    # MCP 도구 핸들러
    def get_task(self, worker_id: str) -> TaskAssignment: ...
    def submit_task(self, task_id, commit_sha, results) -> SubmitResponse: ...
```

### StateMachine

상태 전이와 불변 조건을 관리.

```python
class StateMachine:
    def __init__(self, c4_dir: Path, store: StateStore | None = None):
        self._store = store or SQLiteStateStore(c4_dir / "c4.db")  # 기본값

    def transition(self, event: str, actor: str) -> ProjectStatus:
        # 1. 전이 유효성 검사
        # 2. 불변 조건 확인
        # 3. 상태 변경
        # 4. 이벤트 기록
        # 5. 저장

    def check_gate_conditions(self, config: C4Config) -> str | None:
        # 체크포인트 조건 확인
```

### WorkerManager

워커 생명주기와 stale 워커 복구를 관리.

```python
class WorkerManager:
    def register(self, worker_id: str) -> WorkerInfo: ...
    def heartbeat(self, worker_id: str) -> None: ...  # 암시적 (모든 MCP 호출 시)
    def recover_stale_workers(
        self, stale_timeout_seconds: int, lock_store: LockStore
    ) -> list[dict]: ...  # 30분 비활성 워커 태스크 회수
```

**Stale 워커 복구**:
- 모든 MCP 도구 호출 시 `_touch_worker()`로 암시적 heartbeat
- SupervisorLoop에서 30분마다 stale 워커 검사
- 비활성 워커의 태스크는 자동으로 pending으로 복귀

### StateStore (Protocol)

상태 영속성을 위한 추상 인터페이스.

```python
class StateStore(ABC):
    @abstractmethod
    def load(self, project_id: str) -> C4State: ...

    @abstractmethod
    def save(self, state: C4State) -> None: ...

    @abstractmethod
    def exists(self, project_id: str) -> bool: ...

    @abstractmethod
    def delete(self, project_id: str) -> None: ...
```

**구현체**:
- `SQLiteStateStore`: SQLite 데이터베이스 (`.c4/c4.db`) - **기본값**, WAL 모드로 멀티워커 동시성 지원
- `LocalFileStateStore`: 파일 기반 (`.c4/state.json`) - 레거시, 단일 워커 전용

### LockStore (Protocol)

분산 잠금을 위한 추상 인터페이스. **SQLite `c4_locks` 테이블이 유일한 권위 있는 소스**.

```python
class LockStore(ABC):
    @abstractmethod
    def acquire_scope_lock(
        self, project_id, scope, owner, ttl_seconds
    ) -> bool: ...

    @abstractmethod
    def release_scope_lock(self, project_id, scope) -> bool: ...

    @abstractmethod
    def refresh_scope_lock(
        self, project_id, scope, owner, ttl_seconds
    ) -> bool: ...
```

**구현체**:
- `SQLiteLockStore`: SQLite 데이터베이스 - **기본값**, 30분 TTL로 자동 만료

### SupervisorBackend (Protocol)

Supervisor 리뷰를 위한 추상 인터페이스.

```python
class SupervisorBackend(ABC):
    @abstractmethod
    def review_checkpoint(
        self, prompt: str, context: dict
    ) -> SupervisorResponse: ...
```

**구현체**:
- `ClaudeCliBackend`: Claude CLI 사용
- `MockBackend`: 테스트용 Mock

### AgentRouter (Phase 4)

도메인 기반 에이전트 자동 선택 및 체이닝.

```python
from c4.supervisor.agent_router import get_recommended_agent, get_chain_for_domain

# 도메인으로 추천 에이전트 조회
config = get_recommended_agent("web-frontend")
# config.primary = "frontend-developer"
# config.chain = ["frontend-developer", "test-automator", "code-reviewer"]
# config.handoff_instructions = "Pass component specs..."

# 체인 조회
chain = get_chain_for_domain("ml-dl")
# ["ml-engineer", "python-pro", "test-automator"]
```

**도메인 → 에이전트 매핑**:

| Domain | Primary Agent | Chain |
|--------|--------------|-------|
| `web-frontend` | `frontend-developer` | frontend → test → reviewer |
| `web-backend` | `backend-architect` | architect → python → test → reviewer |
| `fullstack` | `backend-architect` | backend → frontend → test → reviewer |
| `ml-dl` | `ml-engineer` | ml → python → test |
| `mobile-app` | `mobile-developer` | mobile → test → reviewer |
| `infra` | `cloud-architect` | cloud → deployment |
| `library` | `python-pro` | python → docs → test → reviewer |
| `firmware` | `general-purpose` | general → test |
| `unknown` | `general-purpose` | general → reviewer |

**태스크 타입 오버라이드**:
- `debug` → `debugger`
- `security` → `security-auditor`
- `performance` → `performance-engineer`
- `refactor` → `code-refactorer`
- `test` → `test-automator`

## 설계 원칙

### 1. Protocol 기반 추상화
- 구현이 아닌 인터페이스에 의존
- 테스트 용이성
- 백엔드 교체 가능

### 2. 의존성 주입
```python
# 기본 동작 (SQLite - 멀티워커 지원)
daemon = C4Daemon(project_root)

# 커스텀 Store (레거시 파일 기반)
store = LocalFileStateStore(c4_dir)
daemon = C4Daemon(project_root, state_store=store)
```

### 3. 상태 머신 패턴
- 명시적 상태 전이
- 불변 조건 강제
- 이벤트 소싱

### 4. Backward Compatibility
- 기존 코드 변경 없이 동작
- Optional 파라미터로 확장
