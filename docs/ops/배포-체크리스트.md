# C4 배포 체크리스트

프로덕션 환경에 C4를 배포하기 위한 단계별 가이드입니다.

---

## 전체 배포 흐름

```
1. 사전 준비
   ├── Supabase 프로젝트 생성
   ├── 인프라 환경 준비
   └── 환경 변수 설정

2. 데이터베이스 초기화
   ├── 마이그레이션 적용 (17개 SQL)
   └── RLS 정책 검증

3. C5 Hub 서버 배포 (선택)
   ├── 빌드 및 테스트
   ├── Docker 이미지 배포
   └── 헬스체크

4. C4 클라이언트 배포
   ├── install.sh 실행
   ├── 환경 변수 주입
   └── MCP 서버 검증

5. 모니터링 & 백업
   ├── 로깅 설정
   ├── 메트릭 수집
   └── 백업 자동화
```

## 1단계: 사전 준비 (1시간)

### 1.1 Supabase 프로젝트 생성

```bash
# 1. https://supabase.com 방문
# 2. "New Project" 클릭
# 3. 다음 정보 입력:
#    - Project name: c4-prod (또는 팀 이름)
#    - Database password: (강력한 비밀번호 생성)
#    - Region: 팀 위치에 맞는 리전 선택

# 4. 프로젝트 생성 완료 후 API 키 획득
#    Settings → API → URL, anon key, service_role key 복사
```

**출력 예시**:
```
Project URL:     https://xxxxx.supabase.co
Anon Key:        eyJ...
Service Role:    eyJ...
```

### 1.2 인프라 환경 준비

**온프레미스(Bare Metal)의 경우**:
```bash
# 필수 패키지 설치 (Ubuntu/Debian)
sudo apt update
sudo apt install -y \
  golang-1.22 \
  python3.11 python3-pip \
  postgresql-client \
  git curl jq

# Go 경로 설정
export PATH="/usr/lib/go-1.22/bin:$PATH"
go version  # 1.22+ 확인

# uv 설치
curl -LsSf https://astral.sh/uv/install.sh | sh
```

**Docker를 사용하는 경우**:
```bash
# docker-compose.yml (기본 예시)
version: '3.8'
services:
  c4:
    image: golang:1.22-alpine
    volumes:
      - ./c4:/app
    working_dir: /app
    command: ./install.sh --with-hub
```

## 2단계: 데이터베이스 초기화 (15분)

### 2.1 마이그레이션 적용

```bash
# 1. 저장소 클론 (아직 하지 않았다면)
git clone https://git.pilab.co.kr/pi/cq.git
cd c4

# 2. Supabase CLI 설치
npm install -g supabase  # 또는 curl -fsSL https://cli.supabase.io/install.sh | bash

# 3. Supabase 프로젝트 링크
supabase link --project-ref xxxxx  # Supabase 대시보드에서 Project ID 확인

# 4. 마이그레이션 적용
supabase migration up

# 또는 수동으로 SQL 파일 실행:
cd infra/supabase/migrations
psql postgresql://user:password@db.xxxxx.supabase.co:5432/postgres \
  -f 00001_c4_projects.sql \
  -f 00002_c4_tasks.sql \
  ... (모든 파일)
```

**마이그레이션 파일 목록**:
```

00003_c4_state.sql              # 상태 머신
00004_c4_checkpoints.sql        # 체크포인트
00005_c4_personas.sql           # 페르소나
00006_c4_growth.sql             # 학습 기록
00007_c4_traces.sql             # 추적 정보
00008_c4_lighthouses.sql        # Lighthouse 도구 스펙
00009_c4_documents.sql          # 문서
00010_c4_drive.sql              # C0 Drive 스토리지
00011_drive_storage_update_policy.sql
00012_c4_drive_uploaded_by.sql
00013_c1_context_hub.sql        # C1 메시징
00014_c1_security_fixes.sql     # 보안 수정
00015_c4_knowledge_v2.sql       # C9 지식 저장소
00016_c4_doc_usage.sql          # 문서 사용 통계
00017_c4_doc_chunks.sql         # 문서 청크 저장
```

### 2.2 RLS(Row Level Security) 정책 검증

```bash
# Supabase 대시보드에서 확인
# Authentication → Policies 탭

# 또는 SQL로 검증:
psql postgresql://user:password@db.xxxxx.supabase.co:5432/postgres <<'EOF'
SELECT tablename, policyname FROM pg_policies
WHERE schemaname = 'public'
ORDER BY tablename;
EOF

# 예상 출력 (최소 20+ 정책):
# tablename      | policyname
# ───────────────┼─────────────────────
# c4_projects    | auth_project_owner
# c4_projects    | auth_project_member
# c4_tasks       | auth_task_assign
# ... (더 많음)
```

## 3단계: C5 Hub 서버 배포 (선택사항, 30분)

C5 Hub는 분산 작업 큐를 제공합니다. 단일 개발자는 건너뛰어도 됩니다.

### 3.1 빌드 및 테스트

```bash
# 1. 테스트 실행 (모든 테스트 통과 필수)
cd c5
go test ./...

# 예상: 105 passed

# 2. 프로덕션 바이너리 빌드
CGO_ENABLED=0 GOOS=linux GOARCH=amd64 \
  go build -o c5-hub ./cmd/server

# 3. 바이너리 확인
./c5-hub --version
```

- [ ] 모든 테스트 통과 (105/105)
- [ ] 프로덕션 바이너리 빌드 성공

### 3.2 Docker 이미지 배포

```bash
# Dockerfile 예시
cat > Dockerfile <<'EOF'
FROM golang:1.22-alpine AS builder
WORKDIR /build
COPY . .
RUN go build -o c5-hub ./cmd/server

FROM alpine:latest
RUN apk add --no-cache ca-certificates
COPY --from=builder /build/c5-hub /usr/local/bin/
EXPOSE 8585
ENV C5_PORT=8585 C5_DATA_DIR=/data
CMD ["c5-hub"]
EOF

# 빌드
docker build -t c4-c5-hub:latest .

# 실행
docker run -d \
  --name c5-hub \
  -p 8585:8585 \
  -e C5_API_KEY=your-secret-key \
  -v /data/c5:/data \
  c4-c5-hub:latest

# 헬스체크
curl http://localhost:8585/health
```

**Docker Compose 예시** (`docker-compose.yml`):
```yaml
version: '3.8'
services:
  c5-hub:
    image: c4-c5-hub:latest
    ports:
      - "8585:8585"
    environment:
      C5_PORT: 8585
      C5_API_KEY: ${C5_API_KEY}
      C5_DATA_DIR: /data
    volumes:
      - c5-data:/data
    restart: unless-stopped
    healthcheck:
      test: ["CMD", "curl", "-f", "http://localhost:8585/health"]
      interval: 30s
      timeout: 10s
      retries: 3

volumes:
  c5-data:
```

배포:
```bash
# 환경 변수 설정
export C5_API_KEY=your-secret-key

# 시작
docker-compose up -d

# 로그 확인
docker-compose logs -f c5-hub

# 상태 확인
docker-compose ps
```

## 4단계: C4 클라이언트 배포 (20분)

### 4.1 설치 실행

```bash
# 방법 1: 원라인 설치 (권장)
curl -sSL https://git.pilab.co.kr/pi/cq/raw/main/install.sh | bash

# 방법 2: 저장소에서 설치
cd /opt/cq
./install.sh --with-hub  # C5 Hub 활성화하려면

# 방법 3: 클린 설치 (기존 설정 제거)
rm -rf ~/.c4/ ~/.local/bin/cq ~/.mcp.json
./install.sh
```

**설치 옵션**:
```bash
C4_GLOBAL_INSTALL=y ./install.sh    # 대화형 없이 글로벌 설치
C4_INSTALL_DIR=/opt/cq ./install.sh # 커스텀 디렉토리
--with-hub                           # C5 Hub 지원 활성화
--dry-run                            # 검증만 (실제 설치 안 함)
```

- [ ] install.sh 실행 완료
- [ ] 에러 메시지 없음

### 4.2 환경 변수 주입

```bash
# 방법 1: .env 파일 사용 (권장)
cp .env.example .env
# .env에 프로덕션 값 입력
source .env

# 방법 2: 환경 변수 직접 설정
export C4_SUPABASE_URL=https://xxxxx.supabase.co
export C4_SUPABASE_KEY=eyJ...
export C5_HUB_URL=http://c5-hub.internal:8585

# 방법 3: systemd 서비스 (서버 배포 시)
cat > /etc/systemd/system/c4.service <<'EOF'
[Unit]
Description=C4 MCP Server
After=network.target

[Service]
Type=simple
User=c4
ExecStart=/opt/cq/c4-core/bin/cq
EnvironmentFile=/opt/cq/.env.prod
Restart=on-failure
RestartSec=10

[Install]
WantedBy=multi-user.target
EOF

# 활성화
sudo systemctl enable c4
sudo systemctl start c4
```

### 4.3 MCP 서버 검증

```bash

# 2. Claude Code에서 확인
#    → 개발자 도구 (F12) → MCP Servers
```

### 5.3 백업 자동화

```bash
# Supabase 자동 백업 (대시보드 설정)
# Settings → Backups → Enable automated backups

# 또는 수동 백업 스크립트
cat > backup.sh <<'EOF'
#!/bin/bash
BACKUP_DIR=/backups/c4
mkdir -p $BACKUP_DIR

# PostgreSQL 덤프
pg_dump -h db.xxxxx.supabase.co -U postgres > \
  $BACKUP_DIR/backup-$(date +%Y%m%d-%H%M%S).sql

# 로컬 .c4 디렉토리 백업
tar -czf $BACKUP_DIR/c4-local-$(date +%Y%m%d).tar.gz ~/.c4/

# 오래된 백업 삭제 (30일 이상)
find $BACKUP_DIR -mtime +30 -delete
EOF

# 매일 밤 2시에 실행
(crontab -l 2>/dev/null; echo "0 2 * * * bash /opt/cq/backup.sh") | crontab -
```
