# C4 설치 가이드

이 문서는 C4를 설치하고 Claude Code에서 사용할 수 있도록 설정하는 방법을 안내합니다.

C4는 **Go MCP 서버**(Primary)와 **Python Sidecar**(LSP, Knowledge, GPU)로 구성됩니다.

## 사전 요구사항

### 필수
- **Go 1.22+**: Go MCP 서버 빌드에 필요
- **Python 3.12+**: Sidecar(LSP, Knowledge Store, GPU 스케줄러)에 필요
- **uv**: Python 패키지 관리자 ([설치 가이드](https://docs.astral.sh/uv/getting-started/installation/))
- **Claude Code**: Anthropic의 AI 코딩 어시스턴트

### 선택
- **Node.js 18+ / pnpm / Rust toolchain**: C1 데스크톱 앱 빌드 시 필요

---

## 설치

### 1. 소스 클론

```bash
git clone https://git.pilab.co.kr/pi/c4.git
cd c4
```

### 2. Go 바이너리 빌드

```bash
cd c4-core && go build -o bin/c4 ./cmd/c4 && cd ..
```

빌드 결과: `c4-core/bin/c4` (~12MB)

### 3. Python Sidecar 의존성 설치

```bash
uv sync
```

Go MCP 서버가 필요 시 Python Sidecar를 자동으로 기동합니다. `uv sync`로 의존성만 설치하면 됩니다.

## MCP 서버 설정

### 프로젝트 설정 (기본)

프로젝트 루트의 `.mcp.json`이 이미 Go MCP 서버를 구성하고 있습니다:

```json
{
  "mcpServers": {
    "c4": {
      "command": "/path/to/c4/c4-core/bin/c4",
      "args": ["mcp"],
      "env": {
        "C4_PROJECT_DIR": "."
      }
    }
  }
}
```

> **중요**: `/path/to/c4`를 실제 C4 설치 경로로 변경하세요.

### 글로벌 설정 (모든 프로젝트에서 사용)

`~/.claude.json` 파일에 추가:

```json
{
  "mcpServers": {
    "c4": {
      "command": "/path/to/c4/c4-core/bin/c4",
      "args": ["mcp"]
    }
  }
}
```

### Claude Code 재시작

설정을 적용하려면 Claude Code를 **완전히 종료 후 다시 시작**합니다.

## 설치 확인

Claude Code에서 다음 명령을 실행합니다:

```
/c4-status
```

성공 시 다음과 비슷한 메시지가 표시됩니다:
```
C4 not initialized in this project.
Use /c4-init to initialize.
```

## C1 데스크톱 앱 (선택)

C1은 Multi-LLM 프로젝트 탐색기입니다. 필요 시 별도로 빌드합니다.

```bash
cd c1
pnpm install
cargo tauri dev       # 개발 서버
cargo tauri build     # 프로덕션 빌드
```

필수 요구사항: Node.js 18+, pnpm, Rust toolchain

## 문제 해결

### "MCP server not found" 오류

1. `.mcp.json` 또는 `~/.claude.json`에서 바이너리 경로 확인
2. Go 바이너리가 존재하는지 확인: `ls c4-core/bin/c4`
3. 바이너리가 없으면 다시 빌드: `cd c4-core && go build -o bin/c4 ./cmd/c4`
4. Claude Code 재시작

### Python Sidecar 오류

Go 서버가 Sidecar 기동에 실패할 경우:

1. `uv sync`로 Python 의존성 재설치
2. Python 3.12+ 설치 확인: `python --version`
3. uv 설치 확인: `uv --version`

### Go 빌드 실패

1. Go 1.22+ 설치 확인: `go version`
2. 모듈 의존성 다운로드: `cd c4-core && go mod download`
3. 빌드 오류 확인: `cd c4-core && go build ./...`

### MCP 도구가 작동하지 않음

1. `~/.claude.json` 또는 `.mcp.json`에 c4 MCP 서버가 등록되어 있는지 확인
2. MCP 서버 바이너리 경로가 올바른지 확인
3. Claude Code 완전 재시작

### C1 로그인 문제

Supabase 연결이 필요합니다. `.env` 파일에 `SUPABASE_URL`과 `SUPABASE_KEY`가 설정되어 있는지 확인하세요.

## 업데이트

C4를 최신 버전으로 업데이트하려면:

```bash
cd /path/to/c4
git pull

# Go 바이너리 재빌드
cd c4-core && go build -o bin/c4 ./cmd/c4 && cd ..

# Python 의존성 업데이트
uv sync
```

그 다음 Claude Code를 재시작합니다.

---

## 제거

C4를 제거하려면:

```bash
# 1. ~/.claude.json에서 "c4" MCP 서버 항목 삭제

# 2. (선택) C4 폴더 삭제
rm -rf /path/to/c4
```

## 아키텍처 개요

```
Claude Code → Go MCP Server (stdio, 47 tools)
                ├→ Go native: 상태, 태스크, 파일, git, validation, discovery, artifact
                └→ JSON-RPC proxy → Python Sidecar
                                      ├→ LSP (multilspy, Jedi, tree-sitter)
                                      ├→ Knowledge Store (FTS5 + Vector)
                                      └→ GPU Scheduler
```
