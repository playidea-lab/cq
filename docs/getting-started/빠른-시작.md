# 빠른 시작 (5분)

이 가이드는 C4의 핵심 워크플로우를 빠르게 체험할 수 있도록 도와줍니다.

## 전제 조건

- [설치 가이드](설치-가이드.md)를 완료했다고 가정합니다
- Claude Code가 설치되어 있어야 합니다

## Step 1: 환경 점검

프로젝트 디렉토리에서 먼저 환경 상태를 확인합니다:

```bash
cq doctor
```

출력 예시:
```
[✓] cq binary: v0.33.0
[✓] Claude Code: 설치됨
[✓] MCP 서버: .mcp.json 연결 확인
[✓] .c4/ 디렉토리: 초기화됨
```

문제가 있는 항목은 메시지를 따라 해결한 후 진행합니다.

## Step 2: 인증 및 프로젝트 시작

```bash
cq auth login
```

로그인 후 Claude Code에서 프로젝트를 엽니다:

```bash
cq claude
```

이 명령어는:
- 현재 디렉토리에서 Claude Code를 실행
- `.mcp.json` 기반으로 C4 MCP 서버 자동 연결
- `.c4/` 디렉토리가 없으면 자동 초기화

## Step 3: 상태 확인

Claude Code에서:

```
/status
```

출력 예시:
```
## C4 프로젝트 상태

📊 **상태**: INIT
📋 **큐**: 0 대기 | 0 진행중 | 0 완료
👷 **워커**: 0 활성
```

## Step 4: 계획 수립

기능이나 목표를 설명하고 태스크를 생성합니다:

```
/plan "로그인 기능 구현"
```

이 명령어는 대화형으로 진행하며:
1. **DISCOVERY**: 요구사항 수집
2. **DESIGN**: 아키텍처 설계
3. **PLAN**: 실행 가능한 태스크 목록 생성

## Step 5: 실행

계획이 완료되면 실행을 시작합니다:

```
/run
```

Worker가 자동으로:
1. 다음 태스크 요청
2. 태스크 구현
3. 검증 실행 (lint, test)
4. 결과 제출 후 다음 태스크로 반복

## Step 6: 품질 정제

구현이 끝나면 코드 품질을 정제합니다:

```
/polish
```

수정 사항이 0이 될 때까지 반복합니다.

## Step 7: 마무리

빌드·설치·문서·커밋을 한 번에 처리합니다:

```
/finish
```

---

## 시나리오 예시

### (1) 신기능 개발

```
# Claude Code에서
/plan "사용자 프로필 페이지 추가"
/run
/polish
/finish
```

### (2) 빠른 버그 수정

단일 파일 수정처럼 간단한 작업은 direct 모드로:

```
# Claude Code에서 직접 설명
"auth.go에서 토큰 만료 처리 버그 수정해줘"

# 또는 c4_claim → 수정 → c4_report 흐름 사용
```

### (3) 현재 상태 확인

```
/status
```

진행 중인 태스크, 완료 수, Worker 상태를 한눈에 확인합니다.

## 핵심 명령어 요약

| 명령어 | 설명 | 사용 시점 |
|--------|------|-----------|
| `cq doctor` | 환경 점검 | 시작 전 |
| `cq auth login` | 인증 | 처음 사용 시 |
| `cq claude` | Claude Code 실행 | 매 세션 시작 |
| `/status` | 상태 확인 | 언제든지 |
| `/plan` | 계획 수립 | 기능 개발 시작 시 |
| `/run` | 실행 시작 | 계획 완료 후 |
| `/polish` | 품질 정제 | 구현 완료 후 |
| `/finish` | 마무리 커밋 | polish 완료 후 |

## 워크플로우 다이어그램

```
cq doctor → cq auth login → cq claude
                                │
                                ▼
                          /plan "목표"
                                │
                                ▼
                           /run
                          (반복 실행)
                                │
                                ▼
                          /polish
                         (0 수정까지)
                                │
                                ▼
                          /finish
```

## 선택적: LLM Provider 변경

기본값은 Claude Code 구독을 사용합니다. LLM Gateway(임베딩, `c4_llm_call` 등)를 사용하려면:

```bash
# API 키 저장 (암호화, 히스토리 미노출)
cq secret set openai.api_key
```

```yaml
# .c4/config.yaml
llm_gateway:
  enabled: true
  default: openai
  providers:
    openai:
      enabled: true
      default_model: gpt-4o-mini
```

자세한 내용: [LLM 설정](../user-guide/LLM-설정.md)
