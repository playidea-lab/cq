# 빠른 시작 (5분)

이 가이드는 C4의 핵심 워크플로우를 빠르게 체험할 수 있도록 도와줍니다.

## 전제 조건

- [설치 가이드](설치-가이드.md)를 완료했다고 가정합니다
- Claude Code가 설치되어 있어야 합니다

## Step 1: 프로젝트 초기화

프로젝트 디렉토리에서 Claude Code를 열고:

```
/c4-init
```

이 스킬 하나로:
- `.c4/` 디렉토리 생성 (상태, 설정 저장)
- `config.yaml` 기본값 생성
- MCP 서버 연결 확인

## Step 2: 상태 확인

Claude Code에서:

```
/c4-status
```

출력 예시:
```
## C4 프로젝트 상태

📊 **상태**: INIT
📋 **큐**: 0 대기 | 0 진행중 | 0 완료
👷 **워커**: 0 활성
```

## Step 3: 계획 수립

문서를 분석하고 태스크를 생성합니다:

```
/c4-plan
```

이 명령어는:
1. **DISCOVERY**: 프로젝트 분석, 요구사항 수집
2. **DESIGN**: 아키텍처 설계
3. **PLAN**: 태스크 생성

대화형으로 진행되며, 선호도와 결정사항을 물어봅니다.

## Step 4: 실행 시작

계획이 완료되면 실행을 시작합니다:

```
/c4-run
```

Worker Loop이 자동으로:
1. 다음 태스크 요청
2. 에이전트 라우팅 (도메인별 최적 에이전트 선택)
3. 태스크 구현
4. 검증 실행 (lint, test)
5. 결과 제출
6. 다음 태스크로 반복

## Step 5: 진행 상황 모니터링

언제든지 상태를 확인할 수 있습니다:

```
/c4-status
```

출력 예시:
```
## C4 프로젝트 상태

📊 **상태**: EXECUTE
📋 **큐**: 2 대기 | 1 진행중 | 3 완료

### 진행률
[████████░░░░░░░░░░░░] 50% (3/6)

### 활성 태스크
- T-004: API 엔드포인트 구현 (backend-architect)
```

## Step 6: Checkpoint 리뷰

태스크가 완료되고 체크포인트 조건이 만족되면 자동으로 CHECKPOINT 상태가 됩니다:

```
/c4-checkpoint
```

Supervisor가 변경사항을 리뷰하고:
- **APPROVE**: 다음 단계로 진행
- **REQUEST_CHANGES**: 수정 필요 (새 태스크 추가)
- **REPLAN**: 계획 재수립 필요

## 핵심 명령어 요약

| 명령어 | 설명 | 사용 시점 |
|--------|------|-----------|
| `/c4-status` | 상태 확인 | 언제든지 |
| `/c4-plan` | 계획 수립 | 프로젝트 시작 시 |
| `/c4-run` | 실행 시작 | 계획 완료 후 |
| `/c4-stop` | 실행 중지 | 일시 정지 필요 시 |
| `/c4-validate` | 검증 실행 | 수동 검증 시 |
| `/c4-checkpoint` | 리뷰 처리 | CHECKPOINT 상태에서 |

## 워크플로우 다이어그램

```
┌──────┐   ┌───────────┐   ┌────────┐   ┌──────┐
│ INIT │──▶│ DISCOVERY │──▶│ DESIGN │──▶│ PLAN │
└──────┘   └───────────┘   └────────┘   └──┬───┘
                                           │
                    /c4-plan               │
                                           ▼
┌──────────┐   ┌────────────┐   ┌─────────────┐
│ COMPLETE │◀──│ CHECKPOINT │◀──│   EXECUTE   │◀─┐
└──────────┘   └─────┬──────┘   └──────┬──────┘  │
                     │                 │         │
                     │  /c4-checkpoint │ /c4-run │
                     │                 │         │
                     └─────────────────┴─────────┘
                         (반복)
```

## 선택적: LLM Provider 변경

기본값은 Claude Code 구독을 사용합니다. 다른 Provider를 사용하려면:

```yaml
# .c4/config.yaml
llm:
  model: gpt-4o
  api_key_env: OPENAI_API_KEY
```

자세한 내용: [LLM 설정](../user-guide/LLM-설정.md)
