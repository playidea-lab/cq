# 빠른 시작 (5분)

이 가이드는 C4의 핵심 워크플로우를 빠르게 체험할 수 있도록 도와줍니다.

## 전제 조건

- [설치 가이드](설치-가이드.md)를 완료했다고 가정합니다
- Claude Code에서 C4 MCP 서버가 활성화되어 있어야 합니다

## Step 1: 프로젝트 초기화

프로젝트 디렉토리에서 Claude Code를 열고 다음 명령을 실행합니다:

```
/c4-init
```

이 명령은 다음을 생성합니다:
- `.c4/` 디렉토리 (상태, 이벤트 저장)
- `docs/PLAN.md` (프로젝트 계획 템플릿)

## Step 2: 태스크 정의

`docs/PLAN.md` 파일을 열고 태스크를 정의합니다:

```markdown
# 프로젝트 계획

## 태스크 목록

### T-001: 첫 번째 기능 구현
- **설명**: Hello World 기능 추가
- **범위**: `src/`
- **DoD**:
  - [ ] hello() 함수 구현
  - [ ] 테스트 통과

### T-002: 문서 작성
- **설명**: README 업데이트
- **범위**: `docs/`
- **DoD**:
  - [ ] 사용법 추가
```

## Step 3: 실행 시작

계획을 등록하고 실행을 시작합니다:

```
/c4-plan
```

PLAN → EXECUTE 상태로 전환됩니다:

```
/c4-run
```

## Step 4: 태스크 수행

Worker로 참여하여 태스크를 수행합니다:

```
/c4-worker join
```

Worker Loop이 자동으로:
1. 다음 태스크 요청 (`c4_get_task`)
2. 태스크 구현
3. 검증 실행 (`c4_run_validation`)
4. 결과 제출 (`c4_submit`)

## Step 5: 상태 확인

언제든지 프로젝트 상태를 확인할 수 있습니다:

```
/c4-status
```

출력 예시:
```
## C4 Status
- **Status**: EXECUTE
- **Queue**: 1 pending, 1 in_progress, 0 done
- **Workers**: 1 active
```

## Step 6: Checkpoint 리뷰

모든 태스크가 완료되고 체크포인트 조건이 만족되면 자동으로 CHECKPOINT 상태가 됩니다:

```
/c4-checkpoint
```

Supervisor가 변경사항을 리뷰하고:
- **APPROVE**: 다음 단계로 진행
- **REQUEST_CHANGES**: 수정 필요 (새 태스크 추가)
- **REPLAN**: 계획 재수립 필요

## 핵심 명령어 요약

| 명령어 | 설명 | 사용 상태 |
|--------|------|-----------|
| `/c4-init` | 프로젝트 초기화 | - |
| `/c4-plan` | 계획 등록 | INIT, HALTED |
| `/c4-run` | 실행 시작 | PLAN, HALTED |
| `/c4-worker join` | Worker로 참여 | EXECUTE |
| `/c4-status` | 상태 확인 | 모든 상태 |
| `/c4-checkpoint` | 체크포인트 리뷰 | CHECKPOINT |
| `/c4-stop` | 실행 중지 | EXECUTE |

## 상태 흐름

```
INIT → PLAN → EXECUTE → CHECKPOINT → EXECUTE → ... → COMPLETE
         ↑       ↓           ↓
         └───────┴───────────┘ (재작업/재계획 시)
```
