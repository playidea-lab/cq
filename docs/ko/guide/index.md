# CQ란?

**CQ**는 Claude Code, Cursor, Codex CLI와 함께 작동하는 프로젝트 관리 엔진입니다.

AI 기반 개발에 구조를 추가합니다:

- **태스크**에는 완료 조건(DoD)이 있으며 로컬 SQLite 데이터베이스에서 추적됩니다
- **워커**는 각각 격리된 git 워크트리에서 하나의 태스크를 처리하는 AI 에이전트 인스턴스입니다 (Claude Code, Cursor, Codex CLI)
- **리뷰**는 자동으로 생성됩니다 — 모든 구현 태스크에 리뷰 태스크가 자동 생성됩니다
- **체크포인트**는 다음 단계로 넘어가기 전 phase gate 역할을 합니다
- **지식**은 완료된 모든 태스크에서 자동으로 기록되어 미래 태스크에 활용됩니다

## 동작 방식

```
기능을 설명하면 → /c4-plan이 DoD가 있는 태스크를 생성
               → /c4-run이 워커를 스폰 (태스크당 하나)
               → 각 워커: 구현 → 테스트 → 제출
               → 리뷰어 워커가 결과를 검토
               → 빌드, 테스트, 커밋
```

CQ는 **Model Context Protocol (MCP)**를 사용합니다 — Claude Code가 169개 이상의 도구(`c4_*` 접두사)를 통해 CQ 바이너리와 통신합니다. 기본적으로 ~40개 핵심 도구만 표시되며, `CQ_TOOL_TIER=full`로 전체를 노출할 수 있습니다.

CQ는 요청 크기에 따라 자동 라우팅합니다: **Small** (직접 수정), **Medium** (`/c4-quick`), **Large** (`/pi` → plan → run → finish).

## CQ가 아닌 것

- AI 모델이 아닙니다 — AI 코딩 도구를 오케스트레이션합니다
- 코드 생성기가 아닙니다 — 워커가 코드를 작성하고, CQ가 프로세스를 관리합니다
- 특정 스택에 종속되지 않습니다 — 모든 언어와 프레임워크에서 작동합니다
