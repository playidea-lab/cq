# 예시: 첫 번째 태스크

::: info solo 티어
이 예시는 `solo` 티어에서 동작합니다 — 클라우드 설정 불필요. CQ + AI 코딩 어시스턴트만 있으면 됩니다.
:::

## 시나리오

CSV 파일을 읽어서 요약을 출력하는 스크립트를 만들고 싶습니다: 행 수, 컬럼명, 기본 통계.

아직 코드는 없습니다. 아이디어만 있는 상태입니다.

## Step 1 — 아이디어 설명하기

프로젝트 폴더에서 터미널을 열고 CQ를 시작합니다:

```sh
cq claude   # 또는: cq cursor / cq codex / cq gemini
```

그리고 말합니다:

> **나:** "CSV 파일을 읽어서 요약을 출력하는 Python 스크립트를 만들고 싶어 — 행 수, 컬럼명, 수치형 컬럼의 기본 통계."

CQ가 `/pi` 아이디어 탐색 모드로 진입합니다:

```
💡 아이디어 탐색 모드 — 아직 코드는 없어요. 생각을 정리해봅시다.

알겠습니다. CSV 요약 스크립트네요. 몇 가지 확인할게요:
- 결측값은 어떻게 처리할까요?
- 터미널 출력만? 아니면 파일로도 저장?
- 선호하는 라이브러리가 있나요? (pandas, csv 모듈, polars?)

> pandas 괜찮아, 터미널 출력만, 결측값 처리해줘
```

준비가 되면 계획을 제안합니다:

```
📋 계획 준비됨

태스크: CSV 요약 스크립트 만들기
  - 인자로 전달된 CSV 파일 읽기
  - 출력: 행 수, 컬럼명, 컬럼별 데이터 타입
  - 수치형 컬럼: 평균, 최소, 최대, 결측 수
  - 결측값 있어도 오류 없이 처리

진행할까요?  [Y/n]
```

## Step 2 — 실행하기

`Y`를 입력합니다 (또는 그냥 Enter). CQ가 워커를 스폰합니다:

```
/c4-run

  ● 워커 스폰: CSV 요약 스크립트 만들기
  ✓ 생성: scripts/csv_summary.py
  ✓ 테스트 통과
  ✓ 커밋: feat: add CSV summary script

43초 만에 완료.
```

프로젝트에 동작하는 스크립트가 생겼습니다:

```sh
python scripts/csv_summary.py data/my_data.csv

  행 수:    1,204
  컬럼:     8개  (나이, 소득, 점수, 도시, ...)

  수치형 컬럼:
    나이    → 평균: 34.2  최소: 18  최대: 72  결측: 0
    소득    → 평균: 52400  최소: 18000  최대: 210000  결측: 3
    점수    → 평균: 7.4   최소: 1   최대: 10  결측: 12
```

## 무슨 일이 있었나요?

```
목표를 설명했습니다
  ↓
/pi가 요구사항을 정리했습니다 (30초)
  ↓
/c4-run이 격리된 브랜치에서 워커를 스폰했습니다
  ↓
워커가 코드를 작성하고, 테스트를 실행하고, 커밋했습니다
  ↓
결과가 레포에 — 리뷰 완료, 바로 사용 가능
```

보일러플레이트 없이. 스택오버플로우 복사 없이. 디버깅 루프 없이.
