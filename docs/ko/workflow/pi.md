# /pi (Play Idea)

트리거: `/pi` 또는 키워드: `play idea`, `아이디어`, `ideation`, `브레인스토밍`

## 동작 방식

`/pi`는 `/c4-plan` **이전** 단계인 아이디어 탐색 스킬입니다. 막연한 아이디어가 있고 계획을 세우기 전에 구체화가 필요할 때 사용합니다.

4가지 모드로 진행됩니다:

1. **발산 (Diverge)** — 가능성, 유추, 인접 아이디어를 자유롭게 탐색
2. **리서치 (Research)** — 웹 검색 + 지식 베이스로 실제 맥락과 선행 연구 탐색
3. **수렴 (Converge)** — 구체적이고 실행 가능한 개념으로 좁혀나감
4. **토론 (Debate)** — 반론과 리스크 분석으로 아이디어를 검증

아이디어가 충분히 결정(結晶)되면, `/pi`가 `idea.md`를 작성하고 **자동으로 `/c4-plan`을 호출**합니다 — 별도 조작 불필요.

## 예시

```
/pi
```

CQ가 "어떤 아이디어인가요?"라고 물은 뒤, 발산 → 리서치 → 수렴 → 토론 과정을 거쳐 계획 준비 완료 상태로 이끕니다.

또는 바로 시작할 수도 있습니다:

```
/pi "에디터에 실시간 협업 기능 추가"
```

## 사용 기준

| 상황 | 사용 |
|------|------|
| 요구사항이 명확함 | 바로 `/c4-plan` |
| 아이디어가 막연함, 탐색 필요 | `/pi` → 자동으로 `/c4-plan` 전환 |
| 대안 브레인스토밍 | `/pi` |
| 선행 연구 / SOTA 리서치 | `/pi` (리서치 모드) |

## 출력

- `idea.md` — 수렴된 아이디어 구조화 요약 (목표, 제약, 리스크, 핵심 결정)
- 아이디어를 컨텍스트로 삼아 `/c4-plan` 자동 전환

## /pi 이후

`/c4-plan`이 자동으로 호출됩니다. ideation과 planning 사이에 잠시 멈추고 싶다면 `/pi`를 일찍 종료하고, 준비됐을 때 `/c4-plan`을 수동으로 실행하세요.
