# 워크플로우 개요

C4는 상태 머신(State Machine) 기반으로 프로젝트 진행을 관리합니다.
이 문서는 각 상태와 전이 과정을 상세히 설명합니다.

## 상태 다이어그램

```
                    ┌─────────────────────────────────────┐
                    │                                     │
                    v                                     │
┌──────┐  c4_init  ┌──────┐  c4_run  ┌─────────┐         │
│ INIT │ ───────── │ PLAN │ ──────── │ EXECUTE │ ────────┤
└──────┘           └──────┘          └─────────┘         │
                      ^                   │              │
                      │                   │ gate_reached │
                      │ replan            v              │
                      │              ┌────────────┐      │
                      └───────────── │ CHECKPOINT │ ─────┤
                                     └────────────┘      │
                                          │              │
                    approve / approve_final│              │
                                          v              │
                    c4_stop          ┌──────────┐        │
           ─────────────────────────▶│  HALTED  │────────┘
                                     └──────────┘   c4_run/c4_plan

                    all_done         ┌──────────┐
           ─────────────────────────▶│ COMPLETE │
                                     └──────────┘

                    fatal_error      ┌───────┐
           ─────────────────────────▶│ ERROR │
                                     └───────┘
```

## 상태 설명

### INIT
- **설명**: 프로젝트가 C4로 초기화되기 전 상태
- **진입 조건**: `/c4-init` 실행 전
- **허용 작업**: `init`
- **전이**: `c4_init` → PLAN

### PLAN
- **설명**: 태스크 계획 수립 중
- **진입 조건**: 초기화 후 또는 REPLAN 결정 후
- **허용 작업**: `plan`, `run`, `stop`, `status`
- **주요 활동**:
  - `docs/PLAN.md` 작성/수정
  - 태스크 정의 및 의존성 설정
  - 체크포인트 정의
- **전이**:
  - `c4_run` → EXECUTE
  - `c4_stop` → HALTED

### EXECUTE
- **설명**: Worker들이 태스크를 수행 중
- **진입 조건**: `/c4-run` 실행 후
- **허용 작업**: `status`, `worker join`, `worker submit`, `stop`
- **주요 활동**:
  - Worker Loop 실행 (태스크 수행)
  - 검증 실행 (lint, test)
  - 태스크 제출
- **전이**:
  - `gate_reached` → CHECKPOINT (체크포인트 조건 만족)
  - `all_done` → COMPLETE (모든 태스크 및 체크포인트 완료)
  - `c4_stop` → HALTED
  - `fatal_error` → ERROR

### CHECKPOINT
- **설명**: Supervisor가 변경사항을 리뷰 중
- **진입 조건**: 체크포인트 게이트 조건 만족
- **허용 작업**: `status`
- **주요 활동**:
  - 변경사항 번들 생성
  - Supervisor 리뷰
  - 결정 기록
- **전이**:
  - `approve` → EXECUTE (다음 태스크 진행)
  - `approve_final` → COMPLETE (최종 체크포인트 승인)
  - `request_changes` → EXECUTE (수정 태스크 추가됨)
  - `replan` → PLAN (계획 재수립 필요)
  - `fatal_error` → ERROR

### HALTED
- **설명**: 사용자가 실행을 일시 중지함
- **진입 조건**: `/c4-stop` 실행
- **허용 작업**: `run`, `plan`, `status`
- **전이**:
  - `c4_run` → EXECUTE
  - `c4_plan` → PLAN

### COMPLETE
- **설명**: 모든 작업 완료
- **진입 조건**: 모든 체크포인트 통과
- **허용 작업**: `status`
- **전이**: 없음 (최종 상태)

### ERROR
- **설명**: 복구 불가능한 오류 발생
- **진입 조건**: `fatal_error` 이벤트
- **허용 작업**: `status`
- **전이**: 없음 (수동 개입 필요)

## Worker Loop

EXECUTE 상태에서 Worker는 다음 루프를 실행합니다:

```
┌─────────────────────────────────────────────────────────┐
│                    Worker Loop                          │
├─────────────────────────────────────────────────────────┤
│                                                         │
│   ┌──────────────┐                                      │
│   │ c4_get_task  │ ◄───────────────────────┐           │
│   └──────┬───────┘                         │           │
│          │                                  │           │
│          v                                  │           │
│   ┌──────────────┐                          │           │
│   │  태스크 있음? │                          │           │
│   └──────┬───────┘                          │           │
│          │                                  │           │
│    Yes   │   No                             │           │
│          │    └──────────────┐              │           │
│          v                   v              │           │
│   ┌──────────────┐    ┌────────────┐        │           │
│   │   구현 수행   │    │    대기    │        │           │
│   └──────┬───────┘    └────────────┘        │           │
│          │                                  │           │
│          v                                  │           │
│   ┌──────────────┐                          │           │
│   │c4_run_valid  │                          │           │
│   └──────┬───────┘                          │           │
│          │                                  │           │
│          v                                  │           │
│   ┌──────────────┐                          │           │
│   │  c4_submit   │──────────────────────────┘           │
│   └──────────────┘                                      │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

## Supervisor Loop

CHECKPOINT 상태에서 Supervisor Loop이 실행됩니다:

```
┌─────────────────────────────────────────────────────────┐
│                  Supervisor Loop                        │
├─────────────────────────────────────────────────────────┤
│                                                         │
│   ┌──────────────────────┐                              │
│   │ 체크포인트 번들 로드 │                              │
│   └──────────┬───────────┘                              │
│              │                                          │
│              v                                          │
│   ┌──────────────────────┐                              │
│   │    변경사항 분석     │                              │
│   │  - 수정된 파일       │                              │
│   │  - 검증 결과         │                              │
│   │  - 완료된 태스크     │                              │
│   └──────────┬───────────┘                              │
│              │                                          │
│              v                                          │
│   ┌──────────────────────┐                              │
│   │    리뷰 프롬프트     │                              │
│   │    생성 및 실행      │                              │
│   └──────────┬───────────┘                              │
│              │                                          │
│              v                                          │
│   ┌──────────────────────┐                              │
│   │       결정           │                              │
│   └──────────┬───────────┘                              │
│              │                                          │
│     ┌────────┼────────┬───────────┐                     │
│     v        v        v           v                     │
│  APPROVE  REQUEST   REPLAN    APPROVE                   │
│           CHANGES             FINAL                     │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

## 체크포인트 게이트 조건

체크포인트는 다음 조건이 모두 만족될 때 트리거됩니다:

1. **required_tasks**: 지정된 모든 태스크가 완료됨
2. **required_validations**: 지정된 모든 검증이 통과함

예시:
```yaml
checkpoints:
  - id: CP-001
    required_tasks:
      - T-001
      - T-002
    required_validations:
      - lint
      - unit
```

## 불변 조건 (Invariants)

C4는 다음 불변 조건을 항상 유지합니다:

1. **CHECKPOINT 중 Worker 실행 금지**: 리뷰 중에는 코드 변경 불가
2. **EXECUTE 중 PLAN 수정 금지**: 계획 수정은 PLAN 상태에서만
3. **상태 즉시 저장**: 모든 전이 후 상태 파일 즉시 저장

## 브랜치 전략

C4는 자동으로 Git 브랜치를 관리합니다. 별도 설정 없이 Convention over Configuration 원칙을 따릅니다.

### 브랜치 흐름

```
main (또는 default_branch)
  │
  │  c4_start (PLAN → EXECUTE)
  ▼
c4/{project_id}                    ← 작업 브랜치 (자동 생성)
  │
  ├── c4/w-T-001-0                 ← 태스크 브랜치 (c4_get_task 시 생성)
  ├── c4/w-T-002-0
  └── c4/w-T-003-0
        │
        │  체크포인트 APPROVE
        ▼
c4/{project_id}                    ← 태스크 브랜치들이 작업 브랜치로 머지됨
  │
  │  플랜 완료 (approve_final)
  ▼
main                               ← 작업 브랜치가 main으로 머지됨
```

### 주요 시점

| 시점 | 동작 | 생성되는 브랜치 |
|------|------|----------------|
| `c4_start` | 작업 브랜치 생성 | `c4/{project_id}` |
| `c4_get_task` | 태스크 브랜치 생성 | `c4/w-T-XXX-N` |
| `checkpoint APPROVE` | 완료된 태스크 브랜치 → 작업 브랜치 머지 | - |
| `approve_final` | 작업 브랜치 → main 머지 (또는 PR) | - |

### 설정 옵션

`.c4/config.yaml`에서 브랜치 동작을 커스터마이즈할 수 있습니다:

```yaml
# 기본 브랜치 (기본값: main)
default_branch: main

# 작업 브랜치 (기본값: c4/{project_id})
work_branch: null  # null이면 c4/{project_id} 사용

# 플랜 완료 시 동작 (기본값: merge)
completion_action: merge  # merge | pr | manual
```

### completion_action 옵션

| 값 | 동작 |
|----|------|
| `merge` | 작업 브랜치를 main으로 squash-merge (기본값) |
| `pr` | GitHub PR 생성 (`gh pr create`) |
| `manual` | 아무것도 안 함 (사용자가 수동으로 처리) |

### 예시: PR 생성 모드

```yaml
# .c4/config.yaml
completion_action: pr
```

플랜 완료 시:
1. 작업 브랜치를 origin에 push
2. `gh pr create`로 PR 자동 생성
3. 사용자가 GitHub에서 리뷰 후 머지
