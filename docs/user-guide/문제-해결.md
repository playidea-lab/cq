# 문제 해결 (FAQ & 트러블슈팅)

C4 사용 중 발생할 수 있는 일반적인 문제와 해결 방법을 안내합니다.

## 설치 관련

### MCP 서버가 연결되지 않음

**증상**: `/c4-status` 실행 시 "MCP server not found" 오류

**해결**:
1. `~/.claude.json` 설정 확인
   ```json
   {
     "mcpServers": {
       "c4": {
         "command": "uv",
         "args": ["--directory", "/path/to/c4", "run", "python", "-m", "c4.mcp_server"]
       }
     }
   }
   ```

2. 경로가 올바른지 확인
   ```bash
   ls /path/to/c4/c4/mcp_server.py
   ```

3. 의존성 설치 확인
   ```bash
   cd /path/to/c4
   uv sync
   ```

4. Claude Code 완전 재시작 (창 닫기 후 다시 열기)

### 슬래시 명령어가 보이지 않음

**증상**: `/c4-` 입력 시 자동완성이 안 됨

**해결**:
1. MCP 서버 연결 상태 확인 (위 문제 참조)
2. `.claude/commands/` 디렉토리 확인
3. Claude Code 설정 새로고침

## 상태 관련

### "Invalid transition" 오류

**증상**: 명령 실행 시 "Invalid transition: EXECUTE --[c4_init]--> ?" 오류

**원인**: 현재 상태에서 허용되지 않는 명령 실행

**해결**: 상태별 허용 명령 확인
| 상태 | 허용 명령 |
|------|-----------|
| INIT | `init` |
| PLAN | `plan`, `run`, `stop`, `status` |
| EXECUTE | `status`, `worker`, `stop` |
| CHECKPOINT | `status` |
| HALTED | `run`, `plan`, `status` |

```
/c4-status  # 현재 상태 확인
```

### 상태 데이터베이스 손상

**증상**: "database disk image is malformed" 또는 "Validation error" 오류

**해결**:
1. 데이터베이스 백업
   ```bash
   cp .c4/c4.db .c4/c4.db.backup
   ```

2. SQLite 무결성 검사
   ```bash
   sqlite3 .c4/c4.db "PRAGMA integrity_check;"
   ```

3. 심각한 경우 재초기화 (데이터 손실 주의)
   ```bash
   rm -rf .c4
   # 그 후 /c4-init 실행
   ```

> **Note**: 기존 `state.json` 사용 프로젝트는 자동으로 `c4.db`로 마이그레이션됩니다.

## Worker 관련

### 태스크가 할당되지 않음

**증상**: `/c4-worker join` 후 "No task available" 메시지

**가능한 원인**:
1. **모든 태스크 완료**: `/c4-status`로 큐 확인
2. **의존성 미충족**: 선행 태스크가 완료되지 않음
3. **Scope 잠금**: 다른 Worker가 같은 범위 작업 중

**해결**:
```
/c4-status  # 큐 상태 확인
```

### Scope 잠금 충돌

**증상**: "Scope locked by another worker" 메시지

**원인**: 같은 범위(`src/` 등)를 다른 Worker가 작업 중

**해결**:
1. 다른 Worker 완료 대기
2. 다른 범위 태스크 먼저 수행
3. 잠금 만료 대기 (기본 10분)

**강제 해제** (주의 필요):
```bash
# SQLite에서 잠금 직접 삭제
sqlite3 .c4/c4.db "DELETE FROM c4_locks WHERE scope = 'src/';"
```

### 검증 실패

**증상**: `c4_submit` 후 "Validation failed" 오류

**해결**:
1. 검증 명령 수동 실행
   ```bash
   uv run ruff check
   uv run pytest tests/unit
   ```

2. 오류 수정 후 다시 제출

3. 검증 설정 확인 (`.c4/config.yaml`)

## Checkpoint 관련

### Checkpoint에 진입하지 않음

**증상**: 태스크 완료 후에도 CHECKPOINT로 전환되지 않음

**가능한 원인**:
1. **필수 태스크 미완료**: `required_tasks` 확인
2. **검증 실패**: `required_validations` 확인

**해결**:
```
/c4-status  # 체크포인트 조건 확인
```

`docs/PLAN.md`에서 체크포인트 정의 확인:
```markdown
### CP-001
- required_tasks: [T-001, T-002]  # 모두 완료됐는지?
- required_validations: [lint, unit]  # 모두 통과했는지?
```

### Supervisor 리뷰 실패

**증상**: `/c4-checkpoint` 실행 시 오류

**가능한 원인**:
1. **Claude CLI 미설치**: Supervisor가 Claude CLI 사용
2. **번들 생성 실패**: 변경사항이 없음

**해결**:
1. Claude CLI 설치 확인
   ```bash
   claude --version
   ```

2. 수동 모드로 결정
   ```
   /c4-checkpoint APPROVE "수동 승인"
   ```

## 성능 관련

### 상태 로드가 느림

**증상**: 명령 실행이 오래 걸림

**가능한 원인**: 이벤트 파일이 너무 많음

**해결**:
1. 이벤트 아카이브
   ```bash
   mkdir .c4/events/archive
   mv .c4/events/*.json .c4/events/archive/
   ```

2. SQLite VACUUM 실행
   ```bash
   sqlite3 .c4/c4.db "VACUUM;"
   ```

### 메모리 사용량 높음

**증상**: C4 프로세스 메모리 사용량 증가

**해결**:
1. 데이터베이스 크기 확인
   ```bash
   ls -lh .c4/c4.db
   ```

2. 완료된 태스크 정리 (수동)

## 데이터 복구

### 이전 상태로 복원

**방법**:
1. 이벤트 로그에서 상태 추적
   ```bash
   ls -la .c4/events/
   ```

2. 데이터베이스 백업 사용
   ```bash
   cp .c4/c4.db.backup .c4/c4.db
   ```

### Git과 함께 사용

**권장 사항**:
- `.c4/c4.db` - Git에 포함 (상태 공유) 또는 제외 (로컬만 사용)
- `.c4/events/` - Git에 포함 (이력 보존)
- `.c4/c4.db-wal`, `.c4/c4.db-shm` - Git 제외 (WAL 임시 파일)

`.gitignore`:
```
.c4/c4.db-wal
.c4/c4.db-shm
.c4/runs/
```

## 플랫폼별 제한 사항

### Cursor: 25회 Tool Call 제한

**증상**: `/c4-run` 실행 중 갑자기 멈추고 "continue" 버튼이 나타남

**원인**: Cursor는 Agent 모드에서 25회 tool call 제한이 있습니다 (무한 루프 방지 목적).

**해결 방법**:

1. **수동 Continue** (기본)
   - "continue" 버튼 클릭 또는 "계속" 입력
   - 매 25회마다 반복 필요

2. **MAX 모드 사용** (200회로 확장)
   - Cursor 설정에서 MAX 모드 활성화
   - 비용이 증가할 수 있음

3. **다른 클라이언트 사용** (완전 자동화)
   - **Claude Code**: 제한 없음, 완전 자동화 가능
   - **OpenCode**: 터미널 기반, MCP 지원, 제한 없음
   - **Codex CLI**: OpenAI 기반, MCP 지원

**설정 방법** (다른 클라이언트):

```json
// OpenCode: ~/.opencode/config.json
{
  "mcpServers": {
    "c4": {
      "type": "stdio",
      "command": "c4d",
      "args": ["mcp"]
    }
  }
}
```

```bash
# 프로젝트 루트에서 자동 설정
scripts/codex/setup-mcp.sh --write
```

```toml
# Codex CLI: ~/.codex/config.toml (예시)
[mcp_servers.c4]
command = "/path/to/c4/c4-core/bin/c4"
args = ["mcp", "--dir", "/path/to/c4"]
env = { C4_PROJECT_ROOT = "/path/to/c4" }
```

> **참고**: Cursor CLI에서 MCP 도구 호출 시 버그가 보고되어 있습니다 (2025년 1월 기준). Background Agent도 MCP 자동 호출을 지원하지 않습니다. 향후 업데이트로 개선될 수 있습니다.

### Cursor CLI Headless 모드 (cursor-agent)

**증상**: CI/CD 또는 스크립트에서 Cursor를 자동화하고 싶음

**테스트 결과** (cursor-agent 2026.01.17 기준):

| 기능 | 상태 | 조건 |
|------|------|------|
| 파일 읽기 | ✅ 작동 | 기본 |
| 파일 수정 | ✅ 작동 | `--force` 필요 |
| Bash 명령어 | ✅ 작동 | `--force` 필요 |
| MCP 도구 | ❌ 실패 | `spawn ENOENT` 버그 |

**해결 방법: c4 CLI 직접 호출**

MCP가 작동하지 않아도 bash를 통해 c4 CLI를 호출할 수 있습니다:

```bash
# 설치
curl https://cursor.com/install -fsS | bash
~/.local/bin/cursor-agent login

# 사용 예시
cursor-agent -p --force "c4 status 명령을 실행하고 결과를 분석해줘"
cursor-agent -p --force "c4 plan을 실행해서 태스크를 생성해줘"
```

**주요 옵션**:
- `-p, --print`: 비대화형(headless) 모드
- `--force`: 파일 수정 및 bash 명령 허용
- `--approve-mcps`: MCP 서버 자동 승인 (현재 버그로 작동 안 함)
- `--output-format text|json`: 출력 형식

**MCP 버그 상태**: cursor-agent의 MCP spawn 메커니즘에 버그가 있어 외부 프로세스(uv, python, bash 등)를 실행하지 못합니다. 향후 업데이트로 수정될 예정입니다.

## FAQ

### Q: 여러 프로젝트에서 C4를 사용할 수 있나요?
**A**: 네, 각 프로젝트에서 `/c4-init`을 실행하면 독립적으로 사용할 수 있습니다.

### Q: Worker를 여러 개 실행할 수 있나요?
**A**: 네, SQLite WAL 모드로 멀티 Worker를 안전하게 지원합니다. 각 Worker가 별도 Claude Code 창에서 `/c4-worker`를 실행하면 됩니다.

### Q: 팀원과 함께 사용할 수 있나요?
**A**: 로컬 프로젝트는 SQLite로 지원됩니다. 원격 협업은 Redis/Supabase 백엔드로 확장 가능합니다.

### Q: 체크포인트를 건너뛸 수 있나요?
**A**: 네, `passed_checkpoints`에 체크포인트 ID를 수동으로 추가하면 됩니다. (권장하지 않음)

### Q: 태스크 의존성은 어떻게 작동하나요?
**A**: 선행 태스크가 모두 완료되어야 다음 태스크가 할당됩니다.

### Q: 실패한 태스크를 재시도할 수 있나요?
**A**: 태스크 상태를 `pending`으로 변경하면 다시 할당됩니다.
