# 다중 플랫폼 지원

C4는 Claude Code 외에도 Cursor, Codex CLI, Gemini CLI 등 다양한 플랫폼을 지원합니다.

## 지원 플랫폼

| 플랫폼 | 명령어 | 커맨드 위치 |
|--------|--------|-------------|
| Claude Code | `claude` | `.claude/commands/` |
| Cursor | `cursor .` | `.cursor/commands/` |
| Codex CLI | `codex` | `.codex/agents/` |
| Gemini CLI | `gemini` | `.gemini/` |
| OpenCode | `opencode` | `.opencode/` |

## 플랫폼 선택

### 명령행에서 직접 선택

```bash
c4 --platform cursor    # Cursor로 실행
c4 --platform codex     # Codex CLI로 실행
c4 --platform gemini    # Gemini CLI로 실행
```

### 기본 플랫폼 설정

```bash
# 프로젝트 기본값 설정
c4 config platform cursor

# 글로벌 기본값 설정 (모든 프로젝트)
c4 config platform cursor --global

# 현재 설정 확인
c4 config --show
```

### 설정 우선순위

1. `--platform` 옵션 (최우선)
2. 프로젝트 설정 (`.c4/config.yaml`)
3. 글로벌 설정 (`~/.c4/config.yaml`)
4. 환경 변수 (`C4_PLATFORM`)
5. 기본값 (`claude`)

## 플랫폼 설정

### 커맨드 검증

새 플랫폼을 사용하기 전에 필요한 커맨드가 있는지 확인합니다:

```bash
c4 platforms --validate cursor
```

출력 예시:
```
Platform: cursor

Found commands:
  + c4-status
  + c4-init

Missing commands:
  - c4-plan
  - c4-run
  - c4-checkpoint
  - c4-submit

Run 'c4 platforms --setup cursor' to generate templates
```

### 템플릿 생성

누락된 커맨드의 템플릿을 자동 생성합니다:

```bash
c4 platforms --setup cursor
```

생성된 템플릿은 Claude Code 버전(`.claude/commands/`)을 기반으로 하며,
해당 플랫폼에 맞게 **커스터마이즈가 필요합니다**.

### 지원 플랫폼 목록

```bash
c4 platforms --list
```

## 커맨드 인터페이스 명세

각 플랫폼에서 구현해야 할 커맨드 목록과 명세는 
[c4/platforms/SPEC.md](../../c4/platforms/SPEC.md)를 참조하세요.

### 필수 커맨드

| 커맨드 | 설명 | 복잡도 |
|--------|------|--------|
| `c4-status` | 프로젝트 상태 확인 | 낮음 |
| `c4-init` | 프로젝트 초기화 | 낮음 |
| `c4-plan` | Discovery/Design/Plan 워크플로우 | **높음** |
| `c4-run` | Worker Loop 실행 | **높음** |
| `c4-checkpoint` | 체크포인트 리뷰 | 중간 |
| `c4-submit` | 작업 제출 | 중간 |
| `c4-stop` | 실행 중단 | 낮음 |
| `c4-validate` | 검증 실행 | 낮음 |
| `c4-add-task` | 태스크 추가 | 낮음 |
| `c4-clear` | 상태 초기화 | 낮음 |

### 복잡한 커맨드

`c4-plan`, `c4-run`, `c4-checkpoint`, `c4-submit`은 복잡한 워크플로우를 포함합니다.
이들은 Claude Code 버전을 참고하여 플랫폼별로 직접 구현해야 합니다.

## 플랫폼별 특이사항

### Claude Code

- 기준 구현 (Reference Implementation)
- Hooks 지원 (Stop hook, Bash security hook)
- Accept Edits 모드 필요

### Cursor

- Claude Code와 유사한 마크다운 슬래시 커맨드 지원
- Composer Agent 모드 권장
- `.cursor/commands/` 디렉토리 사용

### Codex CLI

- AGENTS.md frontmatter 형식 사용
- triggers 설정으로 자연어 트리거 지원
- `.codex/agents/` 디렉토리 사용

### Gemini CLI

- 슬래시 커맨드 미지원
- MCP 도구 직접 호출
- `.gemini/` 디렉토리에 참조 문서 배치

## 멀티 워커 지원

모든 플랫폼이 동일한 SQLite DB(`.c4/c4.db`)를 사용하므로,
여러 플랫폼에서 동시에 작업할 수 있습니다:

```bash
# 터미널 1 (Claude Code)
c4 --platform claude
/c4-run

# 터미널 2 (Cursor)
c4 --platform cursor
/c4-run

# 터미널 3 (Codex CLI)
c4 --platform codex
codex> c4_get_task("codex-worker")
```
