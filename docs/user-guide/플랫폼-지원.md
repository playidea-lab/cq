# 다중 플랫폼 지원

C4는 하나의 MCP 서버(`c4-core/bin/cq mcp`)를 여러 AI 클라이언트에서 공통으로 사용합니다.

## 지원 플랫폼

| 플랫폼 | 사용 방식 | 프로젝트 아티팩트 |
|--------|-----------|-------------------|
| Claude Code | Skills + MCP | `.claude/skills/` |
| Codex CLI | Agent 트리거 + MCP | `.codex/agents/` |
| OpenCode | MCP 직접 호출 | `.opencode/` |
| **Cursor** | MCP + AGENTS.md 규칙 | `.cursor/mcp.json` |
| Gemini | MCP 직접 호출 또는 문서 기반 | 클라이언트 설정 파일 |

- **Cursor**: 프로젝트에 `.cursor/mcp.json`이 포함되어 있어 Cursor에서 C4 MCP를 바로 사용할 수 있습니다. 자세한 설정과 제한 사항은 [Cursor 가이드](Cursor-가이드.md)를 참고하세요.

## 공통 전제

모든 클라이언트는 동일한 `.c4/c4.db`를 사용하므로 상태/태스크 큐가 공유됩니다.

## Codex CLI 권장 경로

1. 프로젝트 루트에서 `c4 codex` 실행 (MCP/agents 자동 동기화)
2. Codex에서 `c4_status()` 호출로 연결 확인
3. 필요 시 수동 스크립트 사용: `scripts/codex/setup-mcp.sh --write`, `scripts/codex/install-agents.sh`

상세 운용 시나리오는 [Codex CLI 가이드](Codex-가이드.md)를 참조하세요.

## 멀티 클라이언트 동시 운용 예시

```bash
# 터미널 1: Claude Code
# /c4-run

# 터미널 2: Codex CLI
# c4_status() -> c4_get_task(...) -> c4_submit(...)

# 터미널 3: OpenCode
# c4_get_task(...) -> c4_submit(...)
```
