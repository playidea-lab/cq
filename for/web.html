FE 개발자 — 첫 컴포넌트 가이드 ​

CQ 프로젝트에 프론트엔드 개발자로 합류했을 때 첫 컴포넌트를 빠르게 시작하는 방법.

프로젝트 위치 ​

현재: apps/web/ — Evidence Web (Next.js 기반)

apps/web/
├── app/              Next.js App Router
├── components/       공유 컴포넌트
└── lib/              유틸리티

로컬 개발 시작 ​

bash
cd apps/web
pnpm install
pnpm dev            # localhost:3000

환경변수 설정:

bash
cp .env.example .env.local
# NEXT_PUBLIC_SUPABASE_URL, NEXT_PUBLIC_SUPABASE_ANON_KEY 설정

MCP Tool Schema 조회 ​

UI에서 MCP 도구를 호출할 때 사용하는 스키마는 contracts/mcp-tools/에 있습니다. 수동 편집 금지 (DEC-010) — 소스 코드에서 자동 생성됩니다.

bash
# 5개 핵심 tool 스키마 확인
ls contracts/mcp-tools/
# → cq_edge_declare.json, cq_experiment_register.json, cq_hub_submit.json ...

# 특정 스키마 조회 (예: hub submit)
cat contracts/mcp-tools/cq_hub_submit.json

# 스키마 재생성 (소스 코드 변경 후)
cd apps/backend && go run ./cmd/gen-contracts ..

REST API OpenAPI (CF Worker) ​

bash
# CF Worker 엔드포인트 스펙
cat contracts/rest-api/openapi.yaml

# Swagger UI로 보기
npx @redocly/cli preview-docs contracts/rest-api/openapi.yaml

CF Pages 배포 ​

bash
cd apps/web
# 로컬 빌드 + Cloudflare Pages 배포
pnpm dlx @cloudflare/next-on-pages
pnpm dlx wrangler pages deploy .vercel/output/static --project-name=cq-evidence

배포 환경별 설정은 apps/web/wrangler.toml 참조.

CF Pages 대시보드 설정: project cq-evidence → Settings → Builds & deployments → Root directory 를 apps/web 으로 변경 필요 (수동 UI 작업).
