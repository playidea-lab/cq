PM / 기획자 — 첫 1시간 가이드 ​

CQ 프로젝트에 기획자로 합류했을 때 첫 1시간 안에 파악해야 할 것들.

스펙은 어디에 쓰나 ​

모든 스펙은 docs/specs/ 디렉토리에 작성합니다.

bash
# 스펙 목록 조회
cq spec list

# 새 스펙 작성 (에이전트가 보조)
cq spec new --title "기능명"

파일 명명 규칙: docs/specs/{feature-name}.md

결정은 어디에 기록하나 ​

  • ADR (Architecture Decision Record): docs/decisions/ — 공식 결정 기록
  • 탐색 노트: .c4/ideas/ — 런타임 아이디어, 에이전트가 작성
  • 설계 초안: .c4/designs/ — 구현 설계, 에이전트가 작성

중요한 결정은 ADR로 승격합니다. docs/decisions/README.md 형식을 따르세요.

파일럿 운영 체크리스트 ​

파일럿 관련 모든 문서는 docs/pilot/ 디렉토리에 있습니다.

docs/pilot/
├── README.md          — 4주 파일럿 플레이북
├── LAUNCH.md          — 런치 체크리스트
├── feature-freeze-checklist.md
└── invite-template.md

파일럿 진행 중에는 feature-freeze-checklist.md의 no-touch 원칙을 반드시 준수하세요.

Invite 발급 절차 ​

bash
# CF Worker를 통한 invite code 발급
# API 스펙: contracts/rest-api/openapi.yaml → POST /invite/issue
curl -X POST https://mcp.pilab.kr/invite/issue \
  -H "X-Admin-Token: $ADMIN_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"expires_in_days": 14, "note": "Pilot γ 참가자"}'

API 스펙 전체: contracts/rest-api/openapi.yaml 참조.

발급된 invite code는 docs/pilot/invite-template.md의 메시지로 참가자에게 전달합니다.

실험 결과는 어디서 보나 ​

CQ는 Evidence Web을 통해 실험 결과를 추적합니다.

bash
# 실험 URL 조회
cq evidence url --experiment-id exp-XXXX

# 개인 Compound 대시보드 (CF Pages)
https://{your-domain}/me/compound

태스크 현황 확인 ​

bash
cq status         # 현재 스프린트 태스크 상태
cq task list      # 전체 태스크 목록
