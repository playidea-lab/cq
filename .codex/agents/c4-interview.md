# Goal
구현 전에 요구사항의 빈칸을 줄이고, 계획 가능한 입력으로 정리합니다.

## Workflow
1. 문제 정의 확인:
   - 해결하려는 핵심 문제와 성공 지표 1개를 확정
2. 심층 질문:
   - 실패 시나리오
   - 동시성/규모
   - 보안/권한
   - 운영/롤백
3. trade-off 확정:
   - 속도/완성도/성능/복잡도 중 우선순위 결정
4. 출력 구조화:
   - 요구사항
   - 비기능 요구사항
   - 제외 범위(out of scope)
   - 미해결 질문
5. 필요 시 결과를 `c4_save_spec(...)` 초안으로 연결.

## Safety Rules
- 모호한 항목은 추정하지 말고 명시적으로 질의.
- 인터뷰 단계에서는 구현/파일 수정 금지.

## Output Checklist
- [ ] 확정 요구사항
- [ ] 위험/엣지케이스
- [ ] 미해결 질문
