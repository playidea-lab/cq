Distributed Experiments: Hub Submit → GPU Worker → Results ​

Run training jobs on remote GPU machines from your local machine — no SSH, no port forwarding, no manual log collection.

Scenario: Training a PyTorch model on a remote machine with an RTX 5080 while continuing local development work.

When to Use This Workflow ​

Use the Hub/Worker pattern when:

  • Training takes longer than a few minutes
  • You have a dedicated GPU machine (local or cloud)
  • You want metrics to stream back in real time
  • You need results persisted even if the local session ends

For quick local runs, use your terminal directly.

Prerequisites ​

  • CQ installed on both machines
  • cq serve start running on the GPU machine
  • Both machines authenticated (cq auth login)
  • The GPU machine registered as a worker

Verify the worker is visible:

cq workers
ID                         GPU           Status
pi-System-Product-Name     RTX 5080      online

Step 1: Prepare Your Training Script ​

Instrument your training script with the @key=value metric convention. CQ parses stdout automatically — no SDK needed:

python
# train.py
for epoch in range(epochs):
    loss = train_one_epoch(model, loader, optimizer)
    val_loss, accuracy = evaluate(model, val_loader)

    # CQ captures these automatically from stdout
    print(f"Epoch {epoch+1}/{epochs}  @loss={loss:.4f}  @val_loss={val_loss:.4f}  @acc={accuracy:.4f}")

print(f"Training complete  @final_acc={accuracy:.4f}  @best_loss={best_loss:.4f}")

Any line containing @key=value is parsed and stored. Multiple metrics per line are supported.

Step 2: Submit the Job ​

From your local machine (in Claude Code or terminal):

python
cq_job_submit(
    name="resnet50-imagenet-exp001",
    target_worker="pi-System-Product-Name",
    workdir="/home/pi/git/my-model",
    command="python train.py --config configs/exp001.yaml --epochs 100",
    requires_gpu=True,
    exp_id="exp-001"
)

Important: Always specify target_worker. Without it, any available worker (including your local machine) may pick up the job.

CQ responds with a job ID:

Job submitted: job-3a7f2b
  Target: pi-System-Product-Name
  Command: python train.py --config configs/exp001.yaml --epochs 100
  Status: queued

Step 3: Watch Metrics Stream In ​

Monitor the job from your local machine:

cq job watch job-3a7f2b

Metrics appear in real time as the training script prints them:

You can close the watch at any time. Metrics continue streaming and are stored in the Hub database.

Step 4: Retrieve Results ​

After the job completes, retrieve the stored metrics:

python
cq_job_status(job_id="job-3a7f2b")
{
  "status": "succeeded",
  "metrics": {
    "final_acc": 0.9103,
    "best_loss": 0.2341
  },
  "duration_sec": 3412,
  "worker": "pi-System-Product-Name"
}

Step 5: Transfer Model Checkpoints ​

Checkpoints live on the GPU machine. Use CQ Drive to transfer them:

On the GPU machine (via relay call or SSH):

python
cq_drive_upload(
    file_path="/home/pi/git/my-model/checkpoints/exp001_best.pt",
    key="exp001/best.pt",
    note="ResNet50 ImageNet exp001 — acc=0.9103"
)

On your local machine:

python
cq_drive_download(
    key="exp001/best.pt",
    dest_path="./checkpoints/exp001_best.pt"
)

Drive provides permanent storage — checkpoints survive even if the GPU machine is rebooted or the disk is reformatted.

Running Multiple Experiments in Parallel ​

Submit multiple jobs targeting the same worker. The worker queues them and runs one at a time (or in parallel if configured):

python
configs = ["exp002.yaml", "exp003.yaml", "exp004.yaml"]
job_ids = []

for i, cfg in enumerate(configs, start=2):
    result = cq_job_submit(
        name=f"resnet50-{cfg.replace('.yaml','')}",
        target_worker="pi-System-Product-Name",
        workdir="/home/pi/git/my-model",
        command=f"python train.py --config configs/{cfg}",
        requires_gpu=True,
        exp_id=f"exp-{i:03d}"
    )
    job_ids.append(result["job_id"])
    print(f"Submitted: {result['job_id']} → {cfg}")

Each job streams metrics independently. Track all of them:

cq job list
ID             Name                    Status      Worker
job-3a7f2b     resnet50-exp001         succeeded   pi-System-Product-Name

Architecture ​

Your Laptop (Claude Code)          Relay Server            GPU Machine
┌────────────────────────┐         ┌──────────┐            ┌─────────────────────┐
│  cq_job_submit(...)    │──WSS───►│          │──WSS──────►│ cq worker           │
│                        │         │  Relay   │            │                     │
│  cq job watch <id>     │◄──────── metric stream ────────│ train.py stdout     │
│                        │         │          │            │  @loss=0.23         │
│  cq_drive_download(...)│◄────────│──────────│────────────│  @acc=0.91          │
└────────────────────────┘         └──────────┘            └─────────────────────┘

NAT traversal is handled automatically by the Relay. The GPU machine only needs outbound internet access.

@key=value Reference ​

Metrics are parsed from stdout after each newline. The @ prefix is the only requirement.
