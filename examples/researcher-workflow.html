Researcher Workflow: Plan → Train → Evaluate → Iterate ​

A complete end-to-end example of using CQ as a research assistant — from initial idea to a publishable result, with autonomous iteration between cycles.

Scenario: Improving a 3D medical image segmentation model. Starting DICE score: 0.72. Target: 0.85+.

Overview ​

The researcher workflow has four phases:

PhaseToolWhat Happens
Idea/piBrainstorm approaches, surface prior experiments
Plan/planDefine hypotheses, task breakdown, DoD per experiment
Traincq_job_submitExecute experiments on GPU worker
Evaluate & Iterate/c9-loop or manualCompare results, form next hypothesis

Phase 1: Surface What You Know — /pi ​

Start every research session by loading prior context:

/pi "improve 3D medical segmentation DICE from 0.72 to 0.85"

CQ scans your knowledge base for related experiments and surfaces findings:

[pi] Loading related knowledge...
  Found: "exp-011: nnU-Net baseline — DICE 0.72 on CT liver"
  Found: "exp-019: patch size 128→192 improved DICE by 0.03"
  Found: "insight: deep supervision at decoder stages helps convergence"

[pi] Brainstorming directions:
  1. Architecture — add deep supervision at decoder stages 3 and 4
  2. Data — increase patch size to 192x192x64
  3. Loss — combine DICE + focal loss instead of DICE + cross-entropy
  4. Augmentation — elastic deformation for CT-specific shape variance
  5. Training — cosine LR schedule with 10-epoch warm-up

Pick 2–3 directions to explore in parallel. More directions = more GPU time.

Phase 2: Define the Plan — /plan ​

/plan "run ablation: deep supervision + focal loss + patch size on liver CT"

CQ generates a structured plan with tasks:

[plan] Created 3 tasks:

  T-101  Baseline — nnU-Net 128 patch, DICE+CE loss
         DoD: DICE score logged, checkpoint saved to Drive
         Worker: pi-System-Product-Name

  T-102  + Deep supervision — decoder stages 3+4
         DoD: DICE score logged, delta vs T-101 recorded
         Worker: pi-System-Product-Name

  T-103  + Focal loss — replace CE component
         DoD: DICE score logged, delta vs T-101 recorded
         Worker: pi-System-Product-Name

Review and adjust the plan before running. Change scope, add constraints, or split tasks further.

Phase 3: Implement Code Changes — /run ​

/run 2

Two workers implement the code changes in parallel, each on its own git branch:

[worker-1] Implementing T-102: deep supervision
  Modified: model/decoder.py (+47 lines)
  Modified: training/losses.py (+12 lines)
  Validation: PASS

[worker-2] Implementing T-103: focal loss integration
  Modified: training/losses.py (+28 lines)
  Modified: configs/exp023.yaml (+3 lines)
  Validation: PASS

Both branches merged to main.

Phase 4: Submit Training Jobs ​

Submit each experiment to the GPU worker:

python
# Baseline
job_baseline = cq_job_submit(
    name="liver-seg-baseline-exp020",
    target_worker="pi-System-Product-Name",
    workdir="/home/pi/git/liver-seg",
    command="python train.py --config configs/exp020.yaml",
    requires_gpu=True,
    exp_id="exp-020"
)

# Deep supervision variant
job_ds = cq_job_submit(
    name="liver-seg-deepsup-exp021",
    target_worker="pi-System-Product-Name",
    workdir="/home/pi/git/liver-seg",
    command="python train.py --config configs/exp021.yaml",
    requires_gpu=True,
    exp_id="exp-021"
)

# Focal loss variant
job_focal = cq_job_submit(
    name="liver-seg-focal-exp022",
    target_worker="pi-System-Product-Name",
    workdir="/home/pi/git/liver-seg",
    command="python train.py --config configs/exp022.yaml",
    requires_gpu=True,
    exp_id="exp-022"
)

Training script prints metrics using the @key=value convention:

python
# train.py — metrics auto-captured from stdout
print(f"Epoch {epoch}  @loss={loss:.4f}  @dice={dice:.4f}  @lr={lr:.6f}")
print(f"Best model  @best_dice={best_dice:.4f}  @epoch={best_epoch}")

Phase 5: Evaluate Results ​

After all jobs complete, compare metrics:

python
cq_job_status(job_id=job_baseline["job_id"])
cq_job_status(job_id=job_ds["job_id"])
cq_job_status(job_id=job_focal["job_id"])
Experiment    Change                  DICE    Delta
exp-020       Baseline (128 patch)    0.720   —
exp-021       + Deep supervision      0.751   +0.031
exp-022       + Focal loss            0.744   +0.024

Deep supervision wins. Record the finding:

python
cq_knowledge_record(
    key="exp-021-finding",
    content="Deep supervision at decoder stages 3+4 improved DICE by 0.031 on liver CT. "
            "Effect was additive with patch size increase (exp-019). "
            "Focal loss (exp-022) showed smaller gain; may combine with deep supervision next."
)

Phase 6: Iterate ​

Build on the best result. Combine the top performers:

/pi "combine deep supervision + focal loss + larger patch (192), target DICE 0.85"

CQ incorporates your newly recorded findings:

[pi] Loading related knowledge...
  Found: "exp-019: patch size 192 → +0.030 DICE"
  Found: "exp-021: deep supervision → +0.031 DICE"
  Found: "exp-022: focal loss → +0.024 DICE"
  Found: "insight: effects may be additive"

[pi] Hypothesis: combining all three could reach 0.72 + 0.03 + 0.031 + 0.024 ≈ 0.805
  Risk: diminishing returns, training instability with larger patch
  Next experiment: exp-023 — all three combined, 150 epoch budget

Continue the plan → run → train → evaluate cycle until you hit the target.

Autonomous Iteration with /c9-loop ​

For hands-off iteration, hand off to the Research Loop:

/c9-loop start \
  --goal "DICE above 0.85 on liver CT validation set" \
  --budget 8 \
  --metric dice \
  --direction maximize

The LoopOrchestrator cycles autonomously:

[loop] Cycle 1 — current best: 0.751
  Hypothesis: combine deep supervision + focal loss
  Execute: hub submit exp-023
  Result: @dice=0.781  (+0.030)

[loop] Cycle 2 — current best: 0.781
  Hypothesis: patch size 192 + above combination
  Execute: hub submit exp-024
  Result: @dice=0.823  (+0.042)

[loop] Cycle 3 — current best: 0.823
  Hypothesis: 5-fold cross-validation + ensemble
  Execute: hub submit exp-025 (5 folds)
  Result: @dice=0.862  — goal reached.

Saving Final Results ​

When the target is reached, save everything:

python
# Save checkpoint to Drive
cq_drive_upload(
    file_path="/home/pi/git/liver-seg/checkpoints/exp025_ensemble_best.pt",
    key="liver-seg/exp025_final.pt",
    note="Liver CT segmentation — DICE 0.862, 5-fold ensemble"
)

# Record final experiment summary
cq_knowledge_record(
    key="liver-seg-final-result",
    content="Final liver CT segmentation result: DICE 0.862 (5-fold ensemble). "
            "Key contributors: deep supervision (+0.031), focal loss (+0.024), "
            "patch 192 (+0.030), ensemble (+0.039). Total improvement: +0.142 from baseline."
)
