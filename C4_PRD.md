# C4 PRD (Product Requirements Document)

## 1. 개요

### 1.1 제품명
**C4 (Codex–Claude–Completion Control)**  
※ v0 기준: **Claude-only 기반**, Codex/타 모델은 확장 옵션

### 1.2 한 줄 정의
C4는 **AI 에이전트가 하나의 프로젝트를 계획 → 실행 → 검증 → 완료까지 끊김 없이 수행하도록 만드는 프로젝트 운영 시스템**이다.

### 1.3 문제 정의
- 장기 프로젝트에서 AI와의 대화가 단절됨
- IDE(VSCode 등)에 상주해야만 진행 가능
- 계획과 실행, 리뷰가 분리되지 않아 품질이 흔들림
- 반복 작업(TDD, 리팩토링, 마이그레이션)을 사람이 계속 트리거해야 함

### 1.4 해결 목표
- 프로젝트 단위의 **지속성 있는 AI 작업 흐름** 제공
- 계획/실행/검토를 **명시적 상태머신**으로 분리
- AI를 “도우미”가 아닌 **핵심 개발 주체**로 격상
- 사람이 개입하지 않아도 **밤새/며칠 자동 진행** 가능

## 2. 타깃 사용자 & 페르소나

### 2.1 핵심 사용자
- 개인 개발자 / 연구자
- 소규모 스타트업 R&D 팀
- AI-first 개발 워크플로우를 실험하는 조직

### 2.2 페르소나

#### Leader (시스템)
- 역할: 프로젝트 상태 관리, 작업 할당, 게이트 판단
- 구현체: `c4d` (로컬 데몬)
- 특징: 토큰 0, 규칙 기반

#### Worker (실행자)
- 역할: task 단위 구현, 테스트 통과까지 반복
- 구현체: Claude Code + Ralph Loop
- 특징: 빠른 반복, 실패 허용

#### Supervisor (감독자)
- 역할: 체크포인트에서 승인/반려/재계획
- 구현체: Claude Code headless (`claude -p`)
- 특징: Gate 기반 판단, JSON 출력

---

## 3. 사용자 여정 (High-level)

1. 사용자는 프로젝트 폴더에서 `claude` 실행
2. `/c4 init` → Plan 모드 진입
3. 충분한 대화/검색/토론을 통해 계획 문서 완성
4. `/c4 run` → 실행 시작 (멀티 워커 가능)
5. 워커들이 task 수행 (Ralph Loop)
6. 체크포인트 도달 시 Supervisor 자동 리뷰
7. 모든 Gate 통과 시 `/c4 complete`

## 4. 핵심 기능 요구사항 (FR)

### FR-1. Plan 모드
- 대화 기반 계획 수립
- 결과를 문서(`PLAN.md`, `CHECKPOINTS.md`, `DONE.md`, `todo.md`)로 고정

### FR-2. 실행 루프
- task 단위 자동 할당
- 테스트 실패 시 자동 수정 반복

### FR-3. 멀티 워커 지원
- 여러 터미널에서 워커 join 가능
- scope 기반 충돌 방지

### FR-4. 체크포인트 & Gate
- 명시적 Gate 조건
- 자동/수동 승인 분기

### FR-5. Supervisor 자동화
- Claude Code headless 실행
- JSON 기반 결정 파싱

### FR-6. 상태 조회
- `/c4 status`로 언제든 현재 상황 파악

---

## 5. 비기능 요구사항 (NFR)

### 5.1 안정성
- 세션 종료/터미널 종료에도 작업 지속
- 데몬 재시작 시 상태 복구

### 5.2 비용 효율
- 대기 상태에서 토큰 0 소비
- LLM 호출은 필요 시에만

### 5.3 확장성
- Codex / 타 LLM 감독관 추가 가능
- 원격 워커, CI/CD 연동 가능

### 5.4 투명성
- 모든 상태 변화는 이벤트로 기록
- 재현 가능한 실행 로그

---

## 6. 성공 지표 (KPI)

- 사람이 개입하지 않고 완료된 프로젝트 비율
- 평균 task 완료 시간
- 체크포인트 반려율
- 테스트 통과율
- 수동 개입 횟수 감소율

---

## 7. 범위 정의

### 7.1 v0 포함
- 로컬 프로젝트
- Claude Code 기반
- 단일 리포지토리

### 7.2 v0 제외
- SaaS 형태
- 원격 클러스터 워커
- 비용/예산 자동 제어

## 8. 리스크 & 대응

| 리스크 | 대응 |
|------|------|
| 무한 루프 | max-iteration / gate 강제 |
| 품질 저하 | Supervisor Gate |
| 범위 폭주 | DONE.md 기준 고정 |
| 사용자 통제 상실 | 언제든 `/c4 stop` 가능 |

---

## 9. 장기 비전

- C4 = “AI 프로젝트 운영체제”
- AutoML / Agent OS / Research OS로 확장
- MCP / 외부 에이전트와 연동
- 인간은 **방향과 평가**, AI는 **실행과 반복**
