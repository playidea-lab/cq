# Meeting Notes

회의록 작성 가이드. 회의 전 준비부터 배포까지의 워크플로우.

## 트리거

"회의록", "meeting notes", "미팅 노트", "회의 기록"

## Steps

### 1. 사전 준비

- 회의 목적 1문장으로 정의
- 안건 목록 작성 (예상 소요 시간 포함)
- 이전 회의 액션 아이템 팔로업 준비
- 필요한 사전 자료/데이터 정리
- 참석 필수/선택 인원 구분

### 2. 기본 정보 기록

**필수 항목:**
- 회의명
- 일시 (시작~종료)
- 장소 (온라인이면 플랫폼 + 링크)
- 참석자 (이름 + 소속/역할)
- 불참자 (사유)
- 회의 목적

### 3. 논의 내용 요약

- 안건별로 구분하여 기록
- 발언자의 핵심 의견만 요약 (대화 전문 기록 아님)
- 쟁점이 된 부분은 찬반 의견 모두 기록
- 합의에 이르지 못한 사항은 명시적으로 표기

**요약 원칙:**
- "누가 무엇을 말했다"보다 "무엇이 논의되었고, 결론은 무엇이다"
- 구체적 수치/날짜가 언급되면 정확히 기록
- 감정적 표현 제거, 사실 중심 기록

### 4. 결정사항 정리

- 각 결정사항을 명확한 문장으로 기술
- 결정 배경(왜 이렇게 결정했는가) 간략 기록
- 조건부 결정이면 조건 명시
- 보류 사항은 재논의 일정과 함께 기록

### 5. 액션 아이템 작성

| 항목 | 담당자 | 기한 | 비고 |
|------|--------|------|------|
| 구체적 업무 | 이름 | YYYY-MM-DD | 선행 조건 등 |

**규칙:**
- 모든 액션 아이템에 담당자 + 기한 필수
- "검토한다", "확인한다" 같은 모호한 표현 금지
- 산출물이 있으면 산출물 형태 명시 ("보고서 작성" → "A4 1장 요약 보고서")
- `c4_add_todo`로 태스크 시스템에 등록

### 6. 배포

- 회의 당일 또는 익일 내 배포
- 참석자 + 관련자에게 공유
- 결정사항/액션 아이템을 상단에 배치 (TL;DR)
- 다음 회의 일정 안내 (있으면)
- `c4_artifact_save`로 회의록 아카이빙

## 안티패턴

- 대화를 그대로 받아쓰기 (녹취록이 아니다)
- 결정사항 없이 "논의함"으로만 끝나는 회의록
- 액션 아이템에 담당자 없음 ("누군가 해야 함")
- 액션 아이템에 기한 없음 ("가능한 빨리")
- 일주일 뒤에 배포 (기억이 흐려진 후)
- 참석자 확인 없이 배포 (오기재/누락)
