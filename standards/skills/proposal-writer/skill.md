# Proposal Writer

제안서/입찰서 작성 워크플로우. RFP 분석부터 최종 편집까지 체계적으로 진행한다.

## 트리거

"제안서 작성", "입찰서", "proposal", "RFP 대응"

## Steps

### 1. 기존 문서 분석

- `c4_parse_document`로 RFP/입찰공고 파싱
- 발주처 요구사항, 평가 기준, 배점표 추출
- 필수 제출 서류 목록 정리
- 과거 유사 제안서가 있으면 `c4_knowledge_search`로 검색

### 2. RFP 요구사항 추출

- 기능 요구사항 / 비기능 요구사항 분리
- 평가 항목별 배점 확인 (기술 vs 가격 비율)
- 필수 충족 조건(자격 요건, 인증) 체크리스트 작성
- 불명확한 요구사항은 질의서 초안 작성

### 3. 구조 설계

- RFP 평가 항목 순서에 맞춰 목차 구성
- 각 섹션별 분량 배분 (배점 비율 반영)
- 차별화 포인트 3개 이내로 선정
- 핵심 메시지(win theme) 1문장 정의

### 4. 섹션별 초안 작성

**기술 제안:**
- 현황 분석 → 문제 정의 → 해결 방안 → 기대 효과 순서
- 아키텍처 다이어그램, 일정표, 조직도 포함
- 수치와 근거 기반 서술 (정성적 표현 최소화)

**관리 제안:**
- 프로젝트 관리 방법론, 일정, 인력 투입 계획
- 리스크 관리 방안, 품질 보증 체계

**가격 제안:**
- 항목별 단가 산출 근거 명시
- 공공 사업은 SW사업 대가기준 준수 확인

### 5. 교차 검증

- RFP 요구사항 vs 제안 내용 매핑 (누락 항목 체크)
- 평가 항목별 자체 채점 (목표 점수 대비)
- 용어/수치 일관성 확인 (동일 수치가 다르게 표기되지 않았는지)
- 제출 요건(분량, 형식, 파일 포맷) 최종 확인

### 6. 최종 편집

- 핵심 메시지가 각 섹션에 일관되게 반영되었는지 확인
- 시각 자료(표, 차트, 다이어그램) 품질 점검
- 맞춤법/문법 검수
- PDF 변환 후 레이아웃 깨짐 확인
- `c4_artifact_save`로 최종본 저장

## 안티패턴

- RFP를 읽지 않고 과거 제안서 복붙으로 시작
- 가격만 채우고 기술 제안은 일반론으로 채움
- 평가 배점을 무시한 분량 배분
- 발주처 용어 대신 자사 용어 사용
- 차별화 포인트 없이 "최고의 기술력" 같은 추상적 표현 반복
- 교차 검증 없이 제출 (요구사항 누락)
