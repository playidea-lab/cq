# Paper Review

학술 논문 리뷰 방법론 가이드 (3-pass 기법).

## 트리거

"논문 리뷰", "paper review", "학술 리뷰", "논문 평가", "peer review"

## Steps

### 1. First Pass (5-10분): 개요 파악

빠르게 읽고 논문의 전체 그림을 파악:

- 제목, 초록, 서론 읽기
- 각 섹션 제목만 훑기
- 결론 읽기
- 참고 문헌 훑기 (이미 읽은 것 체크)

**이 단계에서 답할 수 있어야 하는 것:**
- 논문의 주제와 기여는?
- 어떤 문제를 풀려는가?
- 사용한 방법론은?

### 2. Second Pass (30-60분): 상세 독해

수식과 증명은 건너뛰되 핵심 내용을 이해:

- 그림, 다이어그램, 테이블을 주의 깊게 읽기
- 주요 참고 문헌 중 미읽은 것 표시 (나중에 읽기)
- 핵심 주장마다 근거가 있는지 체크

**6축 평가:**

| 축 | 평가 기준 |
|----|----------|
| **Quality** | 연구 설계가 건전한가? 방법론이 적절한가? |
| **Novelty** | 기존 연구와 차별점은? 기여가 충분한가? |
| **Technical** | 수식, 알고리즘이 정확한가? 증명에 빈틈 없는가? |
| **Experimental** | 실험이 주장을 뒷받침하는가? 베이스라인 비교가 공정한가? |
| **Discussion** | 한계를 인지하는가? 미래 연구 방향이 합리적인가? |
| **Presentation** | 글이 명확한가? 그림/표가 효과적인가? |

### 3. Third Pass (1-2시간): 깊은 검증

논문을 "재구현"하듯 읽기:

- 수식을 직접 유도하여 검증
- 실험을 재현하려면 무엇이 필요한지 파악
- 모든 가정과 전제를 식별
- 약점, 빠진 참고 문헌, 잠재적 문제 메모

### 4. 리뷰 작성

```markdown
## Summary
[논문 요약 3-5문장]

## Strengths
- [강점 1]
- [강점 2]

## Weaknesses
- [약점 1: 설명 + 근거]
- [약점 2: 설명 + 근거]

## Questions for Authors
- [저자에게 묻고 싶은 질문]

## Minor Issues
- [오타, 그래프 레이블, 형식 등]

## Overall Assessment
Score: [1-10] / Confidence: [low/medium/high]
Decision: [Accept / Weak Accept / Borderline / Weak Reject / Reject]
```

### 5. 리뷰 원칙

- **건설적으로**: "이건 틀렸다" → "이 부분의 근거가 불충분하다. X를 보완하면 강화될 것"
- **구체적으로**: "실험이 부족하다" → "Table 2에서 X 베이스라인과의 비교가 빠져있다"
- **재현성 중시**: "코드/데이터가 공개되지 않아 검증이 어렵다"는 정당한 비판
- **수식 검증**: 핵심 수식은 직접 유도. 차원 분석만으로도 오류 발견 가능

## 논문 비교 시

같은 분야 논문 비교 시:
- 동일한 데이터셋, 동일한 메트릭, 동일한 전처리 조건으로 비교
- mean ± std (최소 3회 실행)
- 통계적 유의성 테스트 (있으면)

## 안티패턴

- 1회만 읽고 리뷰 작성
- "잘 쓴 논문입니다" 같은 내용 없는 칭찬
- 약점 지적 시 대안 제시 없음
- 자기 논문을 인용하라는 요구
- 결과만 보고 방법론/실험 설계 무시
