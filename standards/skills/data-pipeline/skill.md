# Data Pipeline

데이터 파이프라인 설계 및 구축 가이드.

## 트리거

"데이터 파이프라인", "data pipeline", "ETL", "데이터 수집", "데이터 적재", "배치 처리"

## Steps

### 1. 요구사항 정의

- 데이터 소스: 어디서 오는가? (DB, API, 파일, 스트리밍)
- 데이터 싱크: 어디로 가는가? (DW, 분석 DB, 파일, API)
- 주기: 실시간 / 준실시간(분) / 배치(시간/일)
- 볼륨: 건수, 크기, 증가율
- SLA: 지연 허용치, 데이터 손실 허용 여부

### 2. 스키마 설계

- 소스 스키마 분석 + 목적지 스키마 설계
- 데이터 타입 매핑 (소스 → 싱크)
- 스키마 검증: Pandera, JSON Schema, Protobuf 등
- 버저닝: content hash로 데이터 버전 관리

```python
import hashlib
version = hashlib.sha256(data.tobytes()).hexdigest()[:8]
```

### 3. 파이프라인 구현

**배치:**
```
Extract → Validate → Transform → Load → Verify
```

**스트리밍:**
```
Ingest → Parse → Enrich → Route → Store
```

**원칙:**
- 각 단계는 idempotent (재실행해도 동일 결과)
- 실패 시 재시도 가능 (checkpoint 기반)
- 증분 처리 우선 (전체 재처리는 fallback)

### 4. 데이터 품질 검증

파이프라인 단계마다 검증:

| 검증 | 예시 |
|------|------|
| 완전성 | 레코드 수 소스 vs 싱크 일치 |
| 유일성 | PK 중복 없음 |
| 유효성 | NULL 비율, 범위, 형식 |
| 적시성 | 최신 데이터 타임스탬프 확인 |
| 정합성 | 참조 무결성, 집계값 일치 |

### 5. 모니터링

- 파이프라인 실행 상태 (성공/실패/실행 중)
- 처리 시간 + 레코드 수 (이상치 감지)
- 데이터 품질 메트릭 (NULL률, 중복률 추이)
- 알림: 실패, SLA 초과, 품질 이상

### 6. 운영

- 스케줄링: cron / Airflow / 자체 스케줄러
- 실패 시: 자동 재시도 (최대 3회) → 알림 → 수동 대응
- 백필: 과거 데이터 재처리 프로세스 준비
- 문서화: 데이터 흐름도, 스키마 사전, 운영 runbook

## 안티패턴

- 검증 없이 적재 ("들어온 대로 넣자")
- 전체 재처리만 지원 (증분 없음)
- 에러를 삼키고 부분 적재 (데이터 정합성 파괴)
- 하드코딩된 스키마 (소스 변경 시 즉시 깨짐)
