Pilot γ 런치 Runbook (운영자용) ​

이 문서는 Compound Pilot 4주 체험단을 실제로 개시하는 순간 따라 할 checklist입니다. 참고: README.md (배경·철학), invite-template.md, feature-freeze-checklist.md.

0. 사전 검증 (5분, 모두 이미 완료 상태인지 재확인) ​

  • [x] CF Worker secrets 배포 — ADMIN_TOKEN, SUPABASE_ANON_KEY, SUPABASE_SERVICE_ROLE_KEY 등 (2026-04-21 기준 확인)
  • [x] INVITE_RATE_KV KV 네임스페이스 — wrangler.toml 라인 24-25에 ID 고정
  • [x] OAUTH_KV KV 네임스페이스 — wrangler.toml 라인 17-18
  • [x] Supabase migration 적용 — 20260418100000_invite_codes.sql (cq_invite_codes + redeem_invite_code RPC)
  • [x] Feature freeze CI 가드 — scripts/check-pilot-freeze.sh (.gitlab-ci.yml에 연결됨)
  • [x] v1.74.0 릴리즈 — cq update로 워커 기계들 업그레이드 완료
  • [x] cq pilot report CLI 동작 — T-CP-2

1. 초대 코드 5장 발급 ​

sh
# 1) 필요한 2개 값 준비
export PILOT_ADMIN_TOKEN='<CF Worker의 ADMIN_TOKEN secret>'
export PILOT_ISSUER_USER_ID='<본인의 auth.users UUID>'
# ↑ 본인 UUID 확인:
#   python3 -c 'import json;print(json.load(open("'"$HOME"'/.c4/session.json"))["user"]["id"])'

# 2) 5장 발급 (기본 28일 만료)
bash scripts/pilot-issue-invites.sh 5 28 > /tmp/pilot-codes.jsonl

각 줄 예시:

json
{"code":"ABCD-EFGH-IJKL","expires_at":"2026-05-19T...","issue_url":"https://...cq.pilab.kr/join?code=ABCD-EFGH-IJKL"}

2. 체험자 매핑 기록 (PII 주의) ​

/tmp/pilot-codes.jsonl의 코드를 비공개 노트(예: 개인 Notion 페이지 또는 ~/.c4/pilot-participants.md)에 옮기고, 각 코드 옆에 체험자 핸들을 기록:

ABCD-EFGH-IJKL → @alice (github, BYO-RTX3090)
EFGH-IJKL-MNOP → @bob (telegram, colab Pro+)
...
  • 이 매핑은 git에 커밋하지 않음
  • 공개 채널(팀 slack, telegram public)에 코드 자체를 올리지 않음

3. 초대 메시지 발송 ​

invite-template.md을 복사해서 각 체험자에게 개별 DM:

  • [초대 코드] → 각자 할당된 코드
  • [날짜] → pilot start = 오늘, end = +4주
  • [피드백 채널] → Telegram DM 또는 이메일

4. Pilot 시작 기록 ​

sh
# 오늘 날짜를 PILOT_START로 영구 저장
echo "PILOT_START=$(date +%Y-%m-%d)" >> ~/.c4/pilot-metadata
echo "PILOT_END=$(date -v+28d +%Y-%m-%d 2>/dev/null || date -d '+28 days' +%Y-%m-%d)" >> ~/.c4/pilot-metadata
cat ~/.c4/pilot-metadata

5. Feature Freeze 활성화 ​

이 시점부터 4주간 feature-freeze-checklist.md의 banned 패턴은 CI에서 자동 차단됩니다. 운영자는:

  • nudge/recommend_next/next_experiment 등 키워드 포함 PR → CI reject
  • c4-core/internal/onboarding/ 수정 금지
  • docs/pilot/ 이외 pilot 운영용 파일 추가 금지

긴급 버그 수정(P0/P1)은 [PILOT-OK: P0 <사유>] 태그로 commit message에 명시.

6. 주간 Passive Check (5분, 선택) ​

sh
cq pilot report --since $(cut -d= -f2 <<< "$(grep ^PILOT_START ~/.c4/pilot-metadata)") --out /tmp/pilot-draft.md
less /tmp/pilot-draft.md
  • 드래프트는 절대 체험자에게 공유 금지
  • 4주 동안 운영자 혼자만 봄
  • 결과에 따라 행동하지 않음 — 관찰만

7. Week 4 마감 (pilot end 날짜) ​

sh
PILOT_START=$(cut -d= -f2 <<< "$(grep ^PILOT_START ~/.c4/pilot-metadata)")
PILOT_END=$(cut -d= -f2 <<< "$(grep ^PILOT_END ~/.c4/pilot-metadata)")
cq pilot report --since "$PILOT_START" --until "$PILOT_END" --out docs/pilot/pilot-final.md

verdict:

  • pass: γ 합격 → Phase 4 Stage 4/5 진행 + freeze 해제 + 내부 발표
  • inconclusive: 아무도 2번째 실험 안 돔 → Stage 1 (entry) 재검토
  • fail: 활동은 있지만 γ 미달 → Stage 1 rework 필수

8. Rollback 시나리오 ​

체험 중 심각한 문제 발생 시:

  1. 새 초대 발급 즉시 중지 (pilot-issue-invites.sh 실행 안 함)
  2. 이미 발급한 코드는 Supabase console에서 UPDATE cq_invite_codes SET expires_at=NOW() WHERE used_at IS NULL 로 강제 만료
  3. 이슈를 docs/pilot/ROLLBACK-log.md 에 기록 후 retro
