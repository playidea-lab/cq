## Tauri v2 Patterns

### IPC Commands
```rust
#[tauri::command]
async fn greet(name: String) -> Result<String, String> {
    Ok(format!("Hello, {}!", name))
}

// Register in lib.rs
tauri::Builder::default()
    .invoke_handler(tauri::generate_handler![greet])
```

### Frontend Invocation
```typescript
import { invoke } from '@tauri-apps/api/core';

const greeting = await invoke<string>('greet', { name: 'World' });
```

### Plugin Usage
```rust
// Cargo.toml
tauri-plugin-dialog = "2"
tauri-plugin-fs = "2"

// lib.rs
tauri::Builder::default()
    .plugin(tauri_plugin_dialog::init())
    .plugin(tauri_plugin_fs::init())
```

### Capabilities (default.json)
```json
{
  "identifier": "default",
  "windows": ["main"],
  "permissions": [
    "core:default",
    "dialog:default",
    "fs:read-files"
  ]
}
```

## Best Practices
- Use `Result<T, String>` for simple error handling in commands
- Prefer async commands for I/O operations
- Configure CSP in tauri.conf.json for security
- Use `@tauri-apps/api` for type-safe frontend calls
- Test with `cargo tauri dev` during development
