# Presentation Rules

> 기획서, 보고서, 발표자료, 교육자료 작성을 위한 규칙

---

## 공통 원칙

### 1. 슬라이드 구조

| 원칙 | 설명 |
|------|------|
| **1 슬라이드 1 메시지** | 핵심 내용 하나만 전달 |
| **제목 = 결론** | 제목만 읽어도 내용 파악 |
| **위→아래, 왼→오** | 시선 흐름 고려 |
| **여백 활용** | 빽빽함 피하기 |

```
❌ BAD: "매출 현황"
✅ GOOD: "매출 전년 대비 23% 증가"
```

### 2. 정보 계층화

```
Level 1: 핵심 메시지 (제목)
Level 2: 주요 포인트 (3개 이하)
Level 3: 상세 내용 (필요시)
Level 4: 부록/참고 (별도 슬라이드)
```

### 3. 시각화 원칙

| 데이터 유형 | 추천 차트 |
|------------|----------|
| **추이/변화** | 선 그래프, 영역 차트 |
| **비교** | 막대 그래프 (가로/세로) |
| **구성/비율** | 파이 차트, 도넛 차트 |
| **관계** | 산점도, 버블 차트 |
| **프로세스** | 플로우 차트, 타임라인 |
| **구조** | 조직도, 트리 다이어그램 |

```
❌ BAD: 숫자 나열
   "1분기: 100, 2분기: 120, 3분기: 150, 4분기: 180"

✅ GOOD: 차트 + 핵심 수치 강조
   [막대 차트] + "4분기 최고 실적 180억 달성"
```

### 4. 텍스트 규칙

| 항목 | 권장 |
|------|------|
| **글머리 항목** | 3-5개 |
| **한 줄 길이** | 15자 내외 |
| **폰트 크기** | 제목 32pt+, 본문 18pt+ |
| **문장 형태** | 완전한 문장보다 키워드 |

```
❌ BAD:
• 우리 회사는 지난 분기에 신규 고객 확보를 위해
  다양한 마케팅 캠페인을 진행하였으며, 그 결과
  전년 동기 대비 23%의 매출 성장을 달성하였습니다.

✅ GOOD:
• 신규 고객 확보 캠페인 진행
• 매출 23% 성장 (전년 대비)
```

## 레이아웃 패턴

### 기본 레이아웃

```
┌─────────────────────────────┐
│  제목 (결론형)               │
├─────────────────────────────┤
│                             │
│     핵심 콘텐츠 영역          │
│     (차트/텍스트/이미지)      │
│                             │
├─────────────────────────────┤
│  출처/부연 설명 (작은 글씨)   │
└─────────────────────────────┘
```

### 2분할 레이아웃

```
┌─────────────────────────────┐
│  제목                        │
├──────────────┬──────────────┤
│              │              │
│   텍스트     │    차트      │
│   (40%)      │    (60%)     │
│              │              │
└──────────────┴──────────────┘
```

### 3분할 레이아웃 (비교용)

```
┌─────────────────────────────┐
│  제목                        │
├─────────┬─────────┬─────────┤
│         │         │         │
│  옵션A  │  옵션B  │  옵션C  │
│         │         │         │
└─────────┴─────────┴─────────┘
```

## 유형별 템플릿

### 기획서 (Proposal)

```
1. 표지
   - 프로젝트명, 날짜, 작성자

2. 목차
   - 전체 구성 개요

3. 배경/목적 (Why)
   - 현재 문제점
   - 해결 필요성
   - 기대 효과

4. 제안 내용 (What)
   - 핵심 솔루션
   - 주요 기능/특징
   - 차별점

5. 실행 계획 (How)
   - 단계별 일정
   - 마일스톤
   - 담당자/역할

6. 예산/리소스
   - 비용 산출
   - 필요 인력
   - 기타 자원

7. 리스크 및 대응
   - 예상 리스크
   - 대응 방안

8. 요약/다음 단계
   - 핵심 메시지 반복
   - 의사결정 요청
```

### 보고서 (Report)

```
1. 표지

2. Executive Summary
   - 핵심 결론 3줄
   - 주요 수치

3. 배경/목적
   - 보고 범위
   - 분석 기간

4. 현황 분석
   - 데이터 기반 현황
   - 주요 지표
   - 트렌드

5. 문제점/이슈
   - 발견된 문제
   - 원인 분석

6. 개선 방안
   - 단기 조치
   - 중장기 대책

7. 결론 및 제언
   - 핵심 요약
   - 액션 아이템

8. 부록
   - 상세 데이터
   - 참고 자료
```

### 교육자료 (Training)

```
1. 표지
   - 과정명, 대상, 소요시간

2. 학습 목표
   - 이 과정을 마치면...
   - 구체적 성과 목표

3. 목차/커리큘럼
   - 전체 흐름 안내

4. 개념 설명 (반복)
   ┌─────────────────┐
   │ 개념 소개        │
   │ ↓               │
   │ 예시/사례        │
   │ ↓               │
   │ 실습/퀴즈        │
   │ ↓               │
   │ 정리            │
   └─────────────────┘

5. 실습 가이드
   - Step-by-step
   - 스크린샷/화면

6. 핵심 정리
   - 오늘 배운 것
   - 체크리스트

7. Q&A / FAQ
   - 자주 묻는 질문

8. 참고 자료
   - 추가 학습 링크
   - 관련 문서
```

### 제안서 (Pitch)

```
1. 표지
   - 임팩트 있는 한 줄

2. 문제 정의
   - 고객의 Pain Point
   - 공감 유도

3. 솔루션
   - 우리의 해결책
   - 핵심 가치

4. 작동 방식
   - How it works
   - 데모/스크린샷

5. 시장/경쟁
   - 시장 규모
   - 경쟁 우위

6. 비즈니스 모델
   - 수익 구조
   - 가격 정책

7. 팀 소개
   - 핵심 멤버
   - 역량

8. 요청 사항
   - 투자/협력 제안
   - 다음 단계
```

### 회의 발표 (Meeting)

```
1. 아젠다
   - 오늘 다룰 내용 (3개 이하)

2. 현황 공유
   - 진행 상황
   - 주요 지표

3. 논의 사항
   - 이슈 1 → 의견 → 결정
   - 이슈 2 → 의견 → 결정

4. 액션 아이템
   - 누가, 무엇을, 언제까지

5. 다음 회의
   - 일정
   - 준비 사항
```

## 색상 가이드

### 비즈니스 기본

```css
--primary: #2563EB;     /* 메인 파랑 */
--secondary: #64748B;   /* 보조 회색 */
--accent: #F59E0B;      /* 강조 주황 */
--success: #10B981;     /* 긍정 초록 */
--danger: #EF4444;      /* 부정 빨강 */
--background: #F8FAFC;  /* 배경 */
--text: #1E293B;        /* 본문 */
```

### 사용 원칙

| 용도 | 색상 |
|------|------|
| **제목/강조** | Primary |
| **본문** | Text (진한 회색) |
| **긍정 지표** | Success (초록) |
| **부정 지표** | Danger (빨강) |
| **중립 정보** | Secondary (회색) |
| **CTA/핵심** | Accent (주황) |

## 체크리스트

### 작성 전
- [ ] 목적 명확화 (설득? 보고? 교육?)
- [ ] 대상 파악 (임원? 실무자? 고객?)
- [ ] 핵심 메시지 정의 (한 문장)

### 작성 중
- [ ] 1 슬라이드 1 메시지
- [ ] 제목 = 결론
- [ ] 글머리 5개 이하
- [ ] 차트로 시각화

### 작성 후
- [ ] 흐름 검토 (논리적?)
- [ ] 분량 적절 (10-20장)
- [ ] 오탈자 확인
- [ ] 발표 시뮬레이션
