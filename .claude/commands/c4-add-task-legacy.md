# C4 Add Task

새로운 태스크를 프로젝트 큐에 추가합니다.

## 사용법

```
/c4-add-task [설명]
```

인자가 없거나 부분적으로 제공되면, **대화를 통해 필요한 정보를 수집**합니다.

## Instructions

### 1. 현재 상태 확인

먼저 `mcp__c4__c4_status`를 호출하여:
- 현재 프로젝트 상태 확인
- 기존 태스크 ID 패턴 파악 (T-001, FEAT-01 등)
- 프로젝트 도메인 확인

### 2. 대화형 정보 수집

`$ARGUMENTS`가 비어있거나 불완전하면, 다음 정보를 **대화로** 수집합니다:

#### 필수 정보

| 항목 | 질문 예시 | 수집 방법 |
|------|----------|----------|
| **제목** | "어떤 작업을 추가하시겠습니까?" | 자유 입력 |
| **DoD** | "완료 기준을 알려주세요" | 제안 후 확인 |

#### 선택 정보 (추론 가능)

| 항목 | 추론 방법 | 확인 질문 |
|------|----------|----------|
| **task_id** | 기존 패턴 + 다음 번호 | "T-005로 할까요?" |
| **scope** | 제목에서 추론 | "src/auth/ 범위로 할까요?" |
| **domain** | 키워드 감지 | "web-frontend 도메인 맞나요?" |

### 3. DoD 작성 도움

사용자가 DoD를 모호하게 작성하면, **구체적인 예시를 제안**합니다:

```
사용자: "로그인 되게 해줘"

Claude: "DoD(완료 기준)를 구체화해볼게요:
  1. 이메일/비밀번호 입력 폼 렌더링
  2. 유효한 자격증명 시 JWT 반환
  3. 잘못된 자격증명 시 401 에러

  이 정도면 괜찮을까요?"
```

### 4. 확인 및 추가

모든 정보가 수집되면:

1. 최종 확인:
   ```
   다음 태스크를 추가합니다:
   - ID: T-005
   - 제목: 로그인 페이지 구현
   - DoD: 이메일/비밀번호 인증, JWT 반환
   - 스코프: src/auth/

   맞으면 '확인', 수정할 부분 있으면 말씀해주세요.
   ```

2. `mcp__c4__c4_add_todo` 호출

3. `mcp__c4__c4_status`로 결과 확인

## 대화 예시

### 최소 입력 (권장)

```
사용자: /c4-add-task

Claude: 어떤 작업을 추가하시겠습니까?

사용자: 로그인 페이지 만들어줘

Claude: 로그인 페이지 구현이군요. 완료 기준을 정해볼게요:
  - 이메일/비밀번호 폼 렌더링
  - 유효한 인증 시 JWT 토큰 반환
  - 실패 시 에러 메시지 표시

  이 정도면 될까요? 추가하거나 수정할 부분 있으면 말씀해주세요.

사용자: 소셜 로그인도 추가해줘

Claude: 네, DoD에 소셜 로그인(Google, GitHub) 추가했습니다.

  태스크 정보:
  - ID: T-003 (기존 패턴 따름)
  - 제목: 로그인 페이지 구현
  - DoD: 이메일/비밀번호 + 소셜(Google, GitHub) 인증
  - 스코프: src/pages/login/

  추가할까요?

사용자: 응

Claude: ✅ T-003 태스크를 추가했습니다.
```

### 빠른 입력

```
사용자: /c4-add-task 사용자 프로필 페이지

Claude: 사용자 프로필 페이지 구현이군요.

  DoD 제안:
  - 프로필 정보 표시 (이름, 이메일, 아바타)
  - 프로필 수정 폼
  - 변경사항 저장 시 성공 메시지

  괜찮으면 '확인', 수정할 부분 있으면 말씀해주세요.
```

## DoD 작성 원칙

**좋은 DoD의 조건:**
1. **검증 가능**: "~가 동작한다", "~를 반환한다"
2. **구체적**: 모호한 표현 금지 ("개선", "최적화" ❌)
3. **독립적**: 이 태스크만으로 확인 가능

| ❌ 나쁜 DoD | ✅ 좋은 DoD |
|------------|------------|
| "로그인 구현" | "이메일/비밀번호 입력 시 JWT 반환, 실패 시 401" |
| "테스트 추가" | "auth 모듈 테스트 5개 추가, 커버리지 80% 이상" |
| "버그 수정" | "null 입력 시 빈 배열 반환, 테스트 추가" |
| "성능 개선" | "API 응답 시간 500ms → 100ms 이하" |

## 도메인 자동 감지

제목에서 키워드를 감지하여 도메인을 추천합니다:

| 키워드 | 도메인 |
|--------|--------|
| 페이지, UI, 컴포넌트, React | `web-frontend` |
| API, 서버, 엔드포인트, DB | `web-backend` |
| 모델, 학습, 데이터셋 | `ml-dl` |
| Terraform, K8s, 배포 | `infra` |
| iOS, Android, 앱 | `mobile-app` |
